结论：**你的代码在“概念图/教学模拟”层面是对的，但和真实 Qwen2.5-VL 实现差距不小**。它抓住了主干数据流：`图像/视频 → ViT → PatchMerger → Qwen2.5 LLM → logits`，也写到了动态分辨率、3D patch embedding、window attention、MRoPE、PatchMerger 这些关键词；但如果目标是“尽量贴近真实 Qwen2.5-VL”，需要改不少地方。你的文件本身也说明它是“教学目的”“不是生产级模型”，这一定位是合理的。

## 1. 最主要判断

**符合的部分：**

Qwen2.5-VL 的真实顶层结构确实是 `visual` 视觉塔加 `language_model` 文本解码器，Transformers 实现里 `Qwen2_5_VLModel` 明确包含 `self.visual = Qwen2_5_VisionTransformerPretrainedModel` 和 `self.language_model = Qwen2_5_VLTextModel`。([GitHub][1])

你的代码更像是“把 Qwen2.5-VL 论文和 HF 代码概念手写成一个简化 Transformer”，不是 faithful implementation。尤其是配置数值、特殊 token、MRoPE、视觉窗口注意力、视觉 token 插入 LLM 的方式、PatchMerger 细节、causal mask 逻辑，都和真实实现不同。

### C. 不应该手动拼接 `vision_start_emb` / `vision_end_emb`

你的代码现在是：

```python

```

这在教学图上可以，但不是真实 Qwen2.5-VL 的接入方式。真实模型通常先由 processor/tokenizer 生成包含 `<|vision_start|>`, `<|image_pad|>` 或 `<|video_pad|>`, `<|vision_end|>` 的 `input_ids`，然后模型把 image/video placeholder token 对应位置的 embedding 替换成视觉编码器输出。HF 文档示例也是通过 `AutoProcessor.apply_chat_template(...)` 生成文本、图像、视频混合输入，而不是在模型 forward 里手动前置拼接视觉 token。([Hugging Face][2])

建议改成：

```python

if pixel_values is not None:
    vision_embeds = self.visual(pixel_values, grid_thw)  # [num_visual_tokens, hidden]
    image_mask = (input_ids == self.image_token_id).unsqueeze(-1).expand_as(inputs_embeds)
    inputs_embeds = inputs_embeds.masked_scatter(image_mask, vision_embeds.to(inputs_embeds.dtype))
```

这样比“把视觉 token 永远拼到文本前面”更接近真实模型。

### D. MRoPE 当前基本没生效

你写了 `MRoPE` 类，但主模型 forward 里没有真正生成 `position_ids`，也没有把 MRoPE 传进 LLM decoder。视觉 blocks 也基本没有传入真实的视觉 rotary position embedding。真实实现里，视觉 encoder 会根据 `grid_thw` 生成 rotary embedding，并且会重排 window index；LLM 侧也会使用 3D position ids，文本 token 的 temporal/height/width position 相同，因此退化为 1D RoPE。Transformers 代码里明确说明，视觉部分对 temporal、height、width 三个维度分别应用 rotary embedding，文本部分三组 position index 相同，因此等价于普通 1D RoPE。([GitHub][1])

更关键的是，你的 `_compute_3d_rotary` 写成了：

```python
freq = t_freq + h_freq + w_freq
```

这不符合 Qwen2.5-VL 的实现精神。真实实现不是简单把三维频率相加，而是把 head/channel 维度按 `mrope_section` 分段，分别对应 temporal、height、width；代码里有 `mrope_section`，并通过 `cos.split(mrope_section, dim=-1)` 后按三维重新拼接。([GitHub][1])

建议你先做简化但正确的版本：
**不要三维相加，而是按 channel split。**

### E. Window Attention 实际没有实现，而且 full attention 层被错误加了 causal mask

你的 `VisionAttention._window_attention()` 里计算的是完整的 `q @ k.T`，所以其实还是全局 attention，不是 window attention。更严重的是，你的父类 `Attention.forward()` 对所有走 `super().forward()` 的情况都会调用 `_make_causal_mask()`，没有检查 `is_causal`。这会导致视觉侧 full attention 层被加上 causal mask，视觉 token 只能看前面的 token，这和 ViT 的双向 attention 不符。

真实视觉 encoder 会先计算 `window_index` 和 `cu_window_seqlens`，非 full-attention 层使用窗口序列长度，full-attention 层使用完整 `cu_seqlens`。Transformers 代码中在每层判断 `layer_num in self.fullatt_block_indexes`，如果是 full attention 就用全局 `cu_seqlens`，否则用 `cu_window_seqlens`。([GitHub][1])

最低限度建议改成：

```python
def _make_attention_mask(self, B, N, device, external_mask=None):
    if not self.is_causal:
        mask = torch.ones(B, 1, N, N, device=device, dtype=torch.bool)
    else:
        mask = torch.tril(torch.ones(N, N, device=device, dtype=torch.bool))
        mask = mask.unsqueeze(0).unsqueeze(0).expand(B, 1, N, N)

    if external_mask is not None:
        mask = mask & external_mask.bool()
    return mask
```

然后：

```python
attn = attn.masked_fill(~mask, float("-inf"))
```

### F. `apply_rotary_pos_emb` 有维度 bug

你写的是：

```python
cos = cos[..., : q.shape[-2] // 2]
sin = sin[..., : q.shape[-2] // 2]
```

但 `q` 的 shape 是 `[B, num_heads, N, head_dim]`，`q.shape[-2]` 是序列长度 `N`，不是 head dimension。这里应该用 `q.shape[-1]`。

建议改成：

```python
cos = cos[..., : q.shape[-1]]
sin = sin[..., : q.shape[-1]]
```

或者如果你的 `cos/sin` 只有 `head_dim // 2`，就需要先 repeat/interleave 到 `head_dim`，不能直接和完整 `q` 相乘。

### G. PatchMerger 细节不一致

你的 PatchMerger 是：

```python
RMSNorm(s*s*hidden_dim)
Linear(s*s*hidden_dim -> llm_hidden_dim)
GELU
Linear(llm_hidden_dim -> llm_hidden_dim)
```

真实 HF 实现是：

```python
ln_q = RMSNorm(context_dim)
mlp = Linear(context_dim * s^2 -> context_dim * s^2)
      GELU
      Linear(context_dim * s^2 -> out_hidden_size)
```

也就是说，真实实现先对每个 vision hidden 做 RMSNorm，然后把 `spatial_merge_size^2` 个 patch 拼起来，再 MLP 到 LLM hidden size。([GitHub][1])

建议改成更接近真实版本：

```python
class PatchMerger(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.s = config.spatial_merge_size
        self.context_dim = config.vision_hidden_dim
        self.hidden_size = self.context_dim * self.s * self.s
        self.ln_q = nn.RMSNorm(self.context_dim, eps=1e-6)
        self.mlp = nn.Sequential(
            nn.Linear(self.hidden_size, self.hidden_size, bias=True),
            nn.GELU(),
            nn.Linear(self.hidden_size, config.llm_hidden_dim, bias=True),
        )

    def forward(self, x):
        # x: [seq_len, vision_hidden_dim] after window-reordered grouping
        x = self.ln_q(x)
        x = x.view(-1, self.hidden_size)
        return self.mlp(x)
```

### H. 视觉输入 shape 和真实 HF 实现不同

你的 `VisionPatchEmbed` 接收 `[B, 3, T, H, W]`，这对教学很直观。但 HF 实现中的 `Qwen2_5_VisionPatchEmbed.forward()` 接收的是已经被 processor 展平成 patch 单元的 `hidden_states`，再 reshape 成 `[-1, in_channels, temporal_patch_size, patch_size, patch_size]`，然后过 `Conv3d`。([GitHub][1])

这不是说你必须改。教学版保留 `[B, 3, T, H, W]` 更容易理解。但如果你想贴近真实 Transformers，需要把视觉输入改成：

```python
pixel_values: [num_patches, C * temporal_patch_size * patch_size * patch_size]
grid_thw: [num_images_or_videos, 3]
```

而不是直接传原始 `[B, C, T, H, W]`。

## 3. 建议的修改优先级

按收益排序，我建议这样改：

1. **先改 config 数值和 token id**：这是最容易改、最明显错误的部分。
2. **修掉 attention mask bug**：视觉 full attention 不能 causal；window attention 不能假装 full attention。
3. **让 RoPE/MRoPE 真正接入 forward**：现在写了类但基本没用。
4. **把视觉 token 插入方式改成 placeholder replacement**：用 `image_token_id` / `video_token_id` mask 替换 embedding，而不是手动 cat。
5. **PatchMerger 改成真实结构**：`RMSNorm(context_dim) + Linear(hidden_size→hidden_size) + GELU + Linear(hidden_size→out_hidden_size)`。
6. **可选：改成 packed visual input + grid_thw**：如果目标是科研复现/剪枝实验，这一步很重要；如果只是教学演示，可以不改。

## 4. 最终评价

可以这样定位你的代码：

> **适合作为 Qwen2.5-VL 架构教学模拟器，但不适合作为真实 Qwen2.5-VL 的实现或剪枝实验基座。**

如果你后续要做 **Qwen2.5-VL 剪枝研究**，我不建议在这个手写模拟器上做实验。更稳妥的路线是直接基于 Hugging Face 的 `Qwen2_5_VLForConditionalGeneration` 改：剪 `model.visual.blocks`、剪 `model.visual.merger`、剪 `model.language_model.layers`，或者在 processor 之后对 visual tokens 做 token pruning。这样实验结果才有说服力。
