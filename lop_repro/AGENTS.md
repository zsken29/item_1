# LOP 复现项目 AGENTS.md

## 环境
- 使用 Conda 环境 `cv_env`
- Python: `C:\Users\ZSQ\anaconda3\envs\cv_env\python.exe`

## 工作范围
- 当前目录为工作区根目录
- 新增文件默认放在当前工作区内，不写入用户主目录、系统目录或工作区外路径

## 协作原则
- 先理解现有文件和目录结构再修改
- 保持改动聚焦，避免无关重构
- 不覆盖、回滚或删除用户已有改动
- 先定位根因，再做最小且完整的修改

## 编辑规范
- 使用清晰简洁的命名，UTF-8 编码
- 注释只解释非显而易见的逻辑
- 不引入新框架或依赖
- 不写兜底逻辑和宽泛 try/except
- 修改必须完整收口：删除过时入口、重复逻辑、死代码

## Git 规范
- 主动管理 git（提交、分支）无需等待指令
- 不提交模型权重、数据集、缓存
- 提交前查看 `git status --short` 区分改动来源
- 修改完成后汇报涉及文件和验证结果

## 验证
- 修改代码后运行最小但有效的验证
- 用中文回复，最终回复简短说明完成内容和改动文件
- 引用文件使用绝对路径
