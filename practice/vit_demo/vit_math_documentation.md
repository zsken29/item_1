## 3. Patch Embedding 计算

### 3.1 Conv2d

```
Conv2d(3, 6, kernel_size=2, stride=2)
权重形状: [6, 3, 2, 2]，全部初始化为 1.0
偏置形状: [6]，全部初始化为 0
```

**Patch 划分**：

```
4×4 图片分割为 2×2 = 4 个 patches

┌─────────┬─────────┐
│ Patch 0 │ Patch 1 │
│ (左上)   │ (右上)   │
├─────────┼─────────┤
│ Patch 2 │ Patch 3 │
│ (左下)   │ (右下)   │
└─────────┴─────────┘
```

**Patch 0 卷积计算（左上角 2×2 区域）**：

```
Patch 0 包含的像素位置：

通道 0: 位置 [0:2, 0:2] = [[1,2],[5,6]]
通道 1: 位置 [0:2, 0:2] = [[17,18],[21,22]]
通道 2: 位置 [0:2, 0:2] = [[33,34],[37,38]]

计算（权重全 1）：
  通道 0 之和 = 1 + 2 + 5 + 6 = 14
  通道 1 之和 = 17 + 18 + 21 + 22 = 78
  通道 2 之和 = 33 + 34 + 37 + 38 = 142
  输出 = 14 + 78 + 142 = 234

因为所有权重全 1，且偏置全 0，输出通道 0~5 都是 234
```

**Patch 0 最终输出**: `[234, 234, 234, 234, 234, 234]`

**同理计算其他 Patches**：

## 6. Multi-Head Attention

### 6.1 QKV 投影

因为 QKV 权重设置为单位矩阵，Q = K = V = x

### 6.2 注意力计算

由于所有 token 向量完全相同（每个维度都是均匀递增的序列），
每个 head 的 Q @ K^T 得到相同的注意力分数。

### 6.3 Softmax

均匀注意力分布：每个 attn[i,j] = 1/5 = 0.2

### 6.4 加权求和

由于注意力均匀分布，输出 = 均值 of V

## 9. 关键公式

| 步骤 | 公式 |
|------|------|
| Patch Embedding | `y = Conv2d(x)`, `y.flatten(2).transpose(1,2)` |
| CLS Token | `torch.cat([cls_token, x], dim=1)` |
| 位置编码 | `x = x + pos_embed` |
| QKV 投影 | `qkv = W_qkv @ x + b_qkv` |
| 注意力 | `attn = (q @ k^T) / √d_k` |
| Softmax | `attn[i,j] = exp(attn[i,j]) / Σ exp(attn[i,:])` |
| 加权输出 | `y = attn @ v` |
| LayerNorm | `y = γ(x-μ)/√(σ²+ε) + β` |
| GELU | `y ≈ 0.5x(1+tanh(√(2/π)(x+0.0447x³)))` |
| 残差 | `y = x + sublayer(x)` |
