# DivPrune：面向大型多模态模型的基于多样性的视觉Token剪枝 — 详解

**DivPrune: Diversity-aware Visual Token Pruning for Large Multimodal Models**

**作者**: Zhichao Wei, Ruofan Wang, Jingjing Liu, Tianyi Zhou, Lei Cheng, Fanxu Meng
**机构**: 多机构合作
**发表**: ACL 2024 (Findings)
**arXiv**: 2406.12279

---

## ★ Insight（关键洞察）

1. **多样性优先原则**：首次提出"多样性优先于重要性"的剪枝原则，保留多样化的Token比保留"最重要"的Token集合更能保持信息量

2. **长尾内容保护**：低频但独特的视觉内容（如罕见物体）往往被传统方法忽略，多样性剪枝可以有效保护这些长尾内容

3. **DI指标设计**：多样性感知重要性（DI）指标同时考虑Token的独立重要性和其对集合整体多样性的贡献

4. **长尾内容精度提升4%**：在长尾分布的视觉内容上，DivPrune相比重要性剪枝提升了4.4%（70.1% vs 66.8%）

5. **两阶段剪枝框架**：结合轻量级预测器和贪婪多样化策略，实现高效且有效的Token剪枝

## 一、论文核心故事线（先读这里）

**一句话总结**：提出DivPrune，通过多样性感知重要性（DI）指标和两阶段剪枝框架，确保被保留的Token覆盖更多样的视觉概念，在长尾内容上取得显著提升。

**核心创新点**：
- 多样性感知的Token重要性（DI）
- 两阶段剪枝框架
- 贪婪多样化选择算法
- 长尾内容保护

**技术路径**：重要性预测 → 多样性建模 → 贪婪选择 → 长尾保护

## 二、研究背景与动机

### 2.1 VLM的Token冗余问题

视觉语言模型处理大量视觉Token：
- LLaVA-1.5：576个视觉Token（24×24 patches）
- 自注意力复杂度：O(N²)
- 导致高内存和计算开销

### 2.2 现有剪枝方法的局限

**基于重要性评分的方法**（ATPM、SaTT、LLaVA-PruMerge）：
- 倾向于保留相似的Token
- 导致被保留Token集合的信息覆盖范围受限
- 长尾内容容易被忽略

**基于聚类的方法**（FastV、MMG-E）：
- 聚类过程引入额外计算开销
- 难以保证每个聚类簇内的代表性Token真正捕获关键视觉信息

### 2.3 核心观察：长尾分布

视觉Token的分布具有显著的长尾特性：

```
Token重要性分布图：

高重要性区域 ███████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
(头部内容)    (罕见但关键的视觉概念)

低重要性区域 ██████████████████████████████████████████████████
(尾部内容)    (大量相似但冗余的Token)
```

例如，在包含人群场景的图像中：
- 高重要性Token：场景主体（人物A、人物B）
- 低重要性Token：背景人物C、人物D、人物E...

**问题**：如果只保留前50%的Token，可能只保留场景主体，导致背景信息完全丢失。

## 三、方法详解（含公式）

### 3.1 问题定义

给定输入图像 $I$ 和视觉编码器 $f_V$，得到视觉Token序列：

$$\mathbf{V} = \{v_1, v_2, ..., v_N\}, \quad |\mathbf{V}| = N$$

**目标**：找到Token子集 $\mathbf{V}' \subset \mathbf{V}$，使得 $|\mathbf{V}'| = M \ll N$，同时最大化下游任务性能。

### 3.2 Token的独立重要性

首先定义Token $v_i$ 的独立重要性 $s_i$，通过轻量级预测器 $p(\cdot)$ 估计：

$$s_i = p(v_i) \in [0, 1]$$

预测器 $p$ 的训练目标：

$$\mathcal{L}_{pred} = \mathbb{E}_{(I, \mathbf{V})} \left[ \sum_i \left\| s_i - \text{AttentionWeight}(v_i) \right\|^2 \right]$$

### 3.3 Token的多样性贡献

给定已选Token集合 $\mathbf{V}_s$，Token $v_i$ 对集合多样性的贡献定义为：

$$\Delta_{div}(v_i | \mathbf{V}_s) = \frac{1}{|\mathbf{V}_s|} \sum_{v_j \in \mathbf{V}_s} \left(1 - \text{sim}(v_i, v_j)\right)$$

其中 $\text{sim}(\cdot, \cdot)$ 是Token特征空间中的余弦相似度：

$$\text{sim}(v_i, v_j) = \frac{v_i \cdot v_j}{\|v_i\| \cdot \|v_j\|}$$

### 3.4 DI指标定义

综合考虑独立重要性和多样性贡献，定义Token $v_i$ 的**多样性感知重要性（DI）**为：

$$DI(v_i | \mathbf{V}_s) = \alpha \cdot s_i + \beta \cdot \Delta_{div}(v_i | \mathbf{V}_s)$$

其中 $\alpha$ 和 $\beta$ 为可学习的权重参数。

### 3.5 两阶段剪枝框架

#### 阶段一：高重要性Token识别

1. 使用轻量级预测器 $p$ 估计每个Token的重要性分数 $s_i$
2. 设定阈值 $\tau$，选择 $s_i > \tau$ 的Token作为高重要性候选

$$\mathbf{V}_{high} = \{v_i \in \mathbf{V} \mid s_i > \tau\}$$

#### 阶段二：贪婪多样化选择

```
输入: 候选集合 V_{high}，目标大小 M
输出: 保留Token集合 V'

1. 初始化: V' = ∅
2. 对于 t = 1 到 M:
   a. 对于每个 v ∈ V_{high} \ V':
      计算 DI(v | V')
   b. 选择DI值最高的Token:
      v* = argmax_{v ∈ V_{high} \ V'} DI(v | V')
   c. 更新: V' = V' ∪ {v*}
3. 返回 V'
```

**复杂度分析**：
- 时间复杂度：$O(M \cdot |\mathbf{V}_{high}| \cdot d)$
- 空间复杂度：$O(|\mathbf{V}| \cdot d)$

## 四、实验设计与结果分析

### 4.1 实验设置

**评估模型**：
- LLaVA-1.5-7B
- LLaVA-1.5-13B
- Vicuna-v1.5-7B

**基准数据集**：
| 数据集 | 任务类型 | 样本数 |
|-------|---------|-------|
| VQAv2 | 视觉问答 | 83k验证 |
| GQA | 视觉推理 | 12k验证 |
| VQA-CE | VQA挑战 | 125k测试 |
| VizWiz | 视觉障碍辅助 | 8k验证 |
| ScienceQA | 科学问答 | 6k验证 |
| POPE | 幻觉检测 | 3k验证 |
| MME | 综合评估 | 2.4k |

**实现细节**：
- 预测器架构：2层MLP，隐藏维度256
- 训练数据：LLaVA-558K指令微调数据
- 训练策略：4×A100 GPU，batch size 64，lr=1e-4
- 剪枝比例：默认保留50% Token
- 阈值 $\tau$：通过验证集确定为0.3

### 4.2 VQAv2数据集结果

| 方法 | LLaVA-7B | LLaVA-13B | 压缩比 |
|------|----------|-----------|-------|
| No-Prune | 78.5% | 79.8% | 1× |
| ATPM | 76.2% | 77.5% | 2× |
| SaTT | 76.8% | 78.1% | 2× |
| LLaVA-PruMerge | 77.4% | 78.6% | 2× |
| EvoPrune | 77.1% | 78.3% | 2× |
| FastV | 75.9% | 77.2% | 2× |
| **DivPrune** | **78.3%** | **79.5%** | 2× |

**关键发现**：
- DivPrune在2×压缩比下，性能几乎与No-Prune持平
- 相比现有最佳方法（LLaVA-PruMerge），DivPrune提升约0.9%

### 4.3 GQA数据集结果

| 方法 | 准确率 | 压缩比 |
|------|--------|-------|
| No-Prune | 62.0% | 1× |
| ATPM | 59.1% | 2× |
| SaTT | 60.3% | 2× |
| LLaVA-PruMerge | 60.8% | 2× |
| **DivPrune** | **61.6%** | 2× |

### 4.4 长尾内容分析

在包含长尾视觉内容的图像上：

| 方法 | 长尾准确率 | 相对下降 |
|------|-----------|---------|
| No-Prune | 71.2% | - |
| ATPM | 63.4% | -10.9% |
| SaTT | 65.1% | -8.6% |
| LLaVA-PruMerge | 66.8% | -6.2% |
| **DivPrune** | **70.1%** | **-1.5%** |

**结论**：DivPrune在长尾内容上的性能下降显著小于现有方法，验证了多样性策略的有效性。

## 六、与相关工作的关系

### 6.1 与重要性剪枝的关系

重要性剪枝（如ATPM、SaTT）：
- 倾向于保留相似的Token
- 长尾内容被忽略

DivPrune的改进：
- 同时考虑重要性和多样性
- 有效保护长尾内容

### 6.2 与聚类方法的关系

聚类方法（如FastV）：
- 聚类引入额外计算开销
- 难以保证代表性

DivPrune的改进：
- 预测器+贪婪选择，计算更高效
- DI指标保证多样性

## 七、方法优缺点深度评析

### 7.1 优点

1. **长尾内容保护**：多样性策略有效保护低频但重要的视觉内容
2. **性能保持优秀**：2×压缩下性能几乎与完整模型持平
3. **可解释性强**：DI分数提供清晰的Token重要性解释
4. **延迟增加小**：仅增加3ms的推理延迟

### 7.2 缺点

1. **预测器训练**：需要额外的训练数据和计算资源
2. **阈值敏感性**：剪枝阈值需要针对不同模型和任务调优
3. **跨模态多样性**：当前仅考虑视觉Token的多样性

### 7.3 潜在改进方向

1. **自适应压缩比**：根据输入图像复杂度动态调整剪枝比例
2. **跨模态多样性**：将多样性建模扩展到视觉-文本场景
3. **端到端训练**：将剪枝模块与大语言模型联合训练
