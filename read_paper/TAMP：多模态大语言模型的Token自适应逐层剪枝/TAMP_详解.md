# TAMP：多模态大语言模型的Token自适应逐层剪枝——详解

## 一、论文概览

| 维度 | 内容 |
|------|------|
| 论文 | Token-Adaptive Multi-layer Pruning for Multimodal Large Language Models |
| 作者 | Zhicheng Duan, Jing Wu, Kaiyan Guo, Huawei Huang 等（北京大学 & 百度） |
| 会议 | ICLR 2025 |
| 核心问题 | 现有视觉Token剪枝方法使用统一剪枝率，无法适应不同样本和不同层的差异化需求 |
| 核心方法 | 剪枝率预测器（PRP）为每个样本的每层学习自适应剪枝率，逐层递进执行剪枝 |
| 关键结果 | 保留50% Token时性能损失<0.3%，推理加速~2x |

## 二、核心问题与动机

### 2.1 统一剪枝的局限性

传统方法（如LLMRazor、SparseLLM等）对所有样本、所有层使用相同剪枝率，存在以下缺陷：

**样本间差异被忽视**：不同图像的信息密度差异巨大。一张纯色背景图可能只需极少的视觉Token就能正确理解；而一张包含大量细节的复杂场景图需要更多Token才能避免信息丢失。统一剪枝无法区分这两种情况。

**层间差异被忽视**：在Transformer的多层架构中，不同层承担不同的语义角色：
- 浅层（低层）：提取局部视觉特征，如边缘、纹理、颜色
- 中层：构建物体部件和空间关系
- 深层（高层）：整合全局语义，与语言模态对齐

统一剪枝率无法适应这种层级特性。例如，若在浅层就大幅剪枝（如80%），可能丢失关键的低级视觉信息，影响后续所有层的表示质量。

### 2.2 TAMP的核心洞察

> **"不同样本在同一层有不同最优剪枝率，同一样本在不同层有不同最优剪枝率。"**

这一洞察直接来源于对MLLM中视觉Tokenattention模式的观察：不同图像在同一层的token重要性分布差异显著，而同一图像在不同层的信息分布也呈现规律性变化。

## 三、方法详解

### 3.1 整体架构

TAMP在标准MLLM（如LLaVA）的基础上，引入了三个新组件：

1. **剪枝率预测器（Pruning Ratio Predictor, PRP）**：核心创新，为每层输出预测剪枝率
2. **重要性评分模块**：基于Attention模式计算每个Token的重要性
3. **逐层递进剪枝机制**：将每层剪枝结果传递至下一层，形成级联

```
图像 → ViT → Token序列 → [PRP预测每层剪枝率] → 分层Token剪枝 → LLM → 回答
                                        ↑
                               [CLS] + 层深度 → PRP
```

### 3.2 剪枝率预测器（PRP）

**输入条件**：
- 样本级条件 $c_s$：来自ViT最后一层的[CLS]Token embedding，反映整张图像的复杂度
- 层条件 $l_i$：当前层深度（归一化），反映该层的语义处理阶段

**输出**：该层应保留的Token比例 $\hat{r}_i \in [0, 1]$

**实现**：轻量级MLP，约几十K参数量（约0.1%的模型总参数），设计为可插拔模块。

**关键设计**：同时注入样本级信息和层信息，使PRP能够建模"样本×层"的二维自适应空间。

### 3.3 重要性评分

获得各层预测剪枝率后，需要决定保留哪些Token。本文使用**Query-Key注意力加权**作为重要性指标：

$$s_j = \sum_k \text{Softmax}(Q_i \cdot K_{i,k})_j$$

含义：Token $j$ 作为Value被各Attention head关注的加权求和分数。分数越高，说明该Token在当前层的语义构建中贡献越大。

另一种可选方案是基于Token的L2范数或注意力熵，但实验表明QK相似度方法效果最佳。

### 3.4 逐层递进（Layer-wise Progressive）剪枝

这是TAMP的一个关键设计选择：**第 $i$ 层的剪枝结果作为第 $i+1$ 层的输入**。

具体来说：
1. 第1层：基于 $V_1$ 和预测比例 $\hat{r}_1$，选出 $N \cdot \hat{r}_1$ 个Token送入第2层
2. 第2层：基于第1层的输出Token和 $\hat{r}_2$，继续剪枝
3. 以此类推，直至最后一层

**优势**：
- 信息流具有方向性：早期层决定保留哪些视觉特征，这些特征被后续层直接使用
- 避免并行剪枝中不同层选择重叠Token的问题（两个层都选同一个Token，但只保留一个）
- 减少冗余：越到深层，Token数量越少，计算量递减

### 3.5 优化目标

$$\mathcal{L} = \mathcal{L}_{\text{VLM}} + \lambda \cdot \mathcal{L}_{\text{sparse}}$$

- **任务损失**：标准的VQA/Captioning交叉熵损失
- **稀疏性损失**：$\mathcal{L}_{\text{sparse}} = -\frac{1}{L} \sum_{i=1}^{L} \hat{r}_i$（鼓励更低的平均剪枝率）
- $\lambda$ 控制稀疏性强度，可根据目标压缩比调节

### 3.6 训练流程

**阶段1：PRP热启动**
- 冻结MLLM主体参数，仅训练PRP
- 使用少量图像-文本对，让PRP学习基本的视觉重要性模式
- 避免PRP在未训练充分时破坏MLLM的能力

**阶段2：联合微调**
- 解冻PRP + MLLM后几层
- 联合优化任务损失 + 稀疏性损失
- 这样可以确保剪枝策略的调整与MLLM的语言理解能力保持同步

## 四、实验分析

### 4.1 主要结果

| 方法 | GQA Acc | VQAv2 | 保留Token | 加速比 |
|------|---------|-------|-----------|--------|
| LLaVA-1.5 | 62.0 | 76.8 | 100% | 1.0x |
| TAMP (50% keep) | 61.8 | 76.5 | 50% | ~2x |
| SparseLLM (50% keep) | 60.1 | 74.9 | 50% | ~2x |
| LLMRazor (50% keep) | 59.8 | 74.5 | 50% | ~2x |

**结论**：在相同Token减少量下，TAMP显著优于基线方法，证明了自适应剪枝率的有效性。

### 4.2 关键消融发现

1. **PRP vs 均匀剪枝**：移除PRP后性能下降1.5-2.0%，确认自适应机制的关键作用
2. **逐层递进 vs 并行剪枝**：逐层递进高0.8%，说明层级信息传递的价值
3. **层条件的重要性**：加入层深度信息后，PRP能够学到"低层多保留、高层可精简"的规律

### 4.3 可视化分析

论文提供了Token剪枝率的热力图。可观察到：
- 简单图像（如纯色背景）在各层均保持较高的剪枝率（更激进的剪枝）
- 复杂图像（如多物体场景）在浅层保持较低剪枝率，深层逐渐降低
- 这与人类视觉认知的层级特性高度吻合

## 五、贡献与创新点总结

| 创新点 | 描述 | 解决的问题 |
|--------|------|-----------|
| 样本-层级双条件自适应 | 同时考虑样本复杂度和层深度的剪枝率预测 | 样本间和层间差异被忽视的问题 |
| 逐层递进Token流 | 将每层剪枝结果传递至下一层 | 保持信息流方向性，减少冗余 |
| 轻量级PRP | MLP架构，约0.1%模型参数 | 引入的计算开销可控 |
| 两阶段训练 | 热启动+联合微调 | 避免PRP破坏MLLM原有能力 |

---

## 六、局限性

1. **PRP计算开销**：尽管PRP很轻量，但每个样本每层都需要运行一次，带来一定的延迟增加
2. **仅视觉Token**：目前仅对视觉Token剪枝，未探索文本Token剪枝的可能性
3. **视频场景**：仅在图像和短视频上验证，复杂视频场景的适用性有待研究
4. **固定剪枝模式**：训练完成后剪枝率固定，无法根据推理时的具体负载动态调整

---

## 七、与其他工作的对比

| 方法 | 剪枝粒度 | 自适应维度 | 核心思想 |
|------|---------|-----------|--------|
| LLMRazor | 统一 | 无 | 基于敏感度的统一剪枝 |
| SparseLLM | 统一 | 无 | 基于熵的统一剪枝 |
| LLM-Pruner | 结构化 | 无 | 神经元级别的结构化剪枝 |
| TAMP | Token级 | 样本×层 | 剪枝率预测器实现双维自适应 |

TAMP的核心差异化在于将"样本自适应"和"层自适应"两个维度结合起来，这是此前工作没有充分探索的方向。

---

## 八、实践建议

1. **适用场景**：需要减少视觉Token计算成本、同时保持MLLM多模态理解能力的应用
2. **实施门槛**：需要额外的训练过程来优化PRP
3. **调参建议**：稀疏性损失权重 $\lambda$ 应根据目标Token保留比例调整
4. **结合使用**：可与知识蒸馏结合，进一步恢复剪枝后的性能损失
