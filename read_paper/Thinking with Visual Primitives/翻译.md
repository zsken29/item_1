# Thinking with Visual Primitives: Holistic Composition of Visual Information

## 视觉原语思维：视觉信息的整体组合

**CVPR 2024**

**作者**: Yike Guo, Xin Li, Feng Tian, Liguang Zhou, Yehui Yang, Yibin Wang, Pi Liu, Hongtao Lu

**机构**: FUNSD

## 摘要

视觉问答（VQA）要求模型能够同时理解视觉信息并执行复杂的推理。然而，现有的视觉-语言模型在组合推理方面存在明显不足，即难以将多个视觉元素正确组合以进行推理。本文认为，这种组合推理能力的缺陷源于缺乏对整体视觉输入进行离散化、可区分原语分解的显式中间视觉表示层。本文提出视觉原语思维（Primitive-of-Thought, PoT），这是一种新颖的思维链范式，能够将复杂的视觉信息分解为离散的视觉原语。原语集合 P = {P_cls, P_attr, P_rel, P_part, P_con} 涵盖了不同粒度层级的视觉信息：类别、属性、关系、部位和概念。基于视觉原语，PoT引导模型先分解复杂视觉输入，再将其重新组合为整体理解。在ShapeWorld、GQA、NLVR2、SEED-Bench和VQAv2等五个基准数据集上的实验表明，PoT显著提升了VLM的组合推理能力。

## 1. 引言

大型语言模型（LLM）在语言推理方面取得了显著进展。通过思维链（Chain-of-Thought, CoT）提示，LLM能够生成中间推理步骤，将复杂问题分解为可管理的子问题。然而，将CoT的成功迁移到视觉领域面临根本性挑战：文本推理步骤和视觉表示本质上是异构的。语言推理步骤是离散的、语义丰富符号，而视觉表示通常是连续的、高维的张量。

视觉问答任务要求模型理解视觉内容并回答问题，这对组合推理能力提出了很高要求——模型需要识别图像中的多个视觉元素，理解它们之间的关系，并基于这些信息进行推理。然而，现有的视觉-语言模型（如LLaVA和MiniGPT-4）在组合推理方面存在系统性缺陷。

为了诊断这些缺陷的根源，本文进行了系统性分析。实验揭示了三个关键问题：

**问题一：属性绑定错误。** 现有VLM倾向于将不同对象的属性混淆。例如，在组合属性理解测试中，模型难以正确区分"红色圆圈"和"黄色三角形"。

**问题二：关系推理错误。** 现有VLM在空间关系和语义关系推理上表现不佳。例如，对于"球在盒子左边"这样的问题，模型容易混淆主客体。

**问题三：视觉-语言不对齐。** 现有VLM过度依赖语言先验，忽视视觉证据。这种"视觉-语言不对齐"导致模型在需要精确视觉理解的任务上失败。

本文假设，这些组合推理缺陷的根源在于缺乏显式的中间视觉表示层。类比CoT通过语言中间步骤促进语言推理，本文提出通过视觉原语提供"视觉工作记忆"，使VLM能够先分解再组合视觉信息。

本文提出视觉原语思维（PoT），核心组件包括：

- **原语提取器（Primitive Extractor, PE）**：从图像中提取5类视觉原语：类别（P_cls）、属性（P_attr）、关系（P_rel）、部位（P_part）和概念（P_con）
- **原语聚合器（Primitive Aggregator, PA）**：将原语聚合成结构化区域图表示
- **答案解码器（Answer Decoder, AD）**：基于结构化表示解码最终答案

PoT的关键设计包括：
1. **双路视觉编码**：使用DINOv2（语义级）和CLIP（像素级）双路编码，确保同时捕获高层语义和细节视觉特征
2. **渐进式解码**：先解码原语序列，再由原语引导最终答案
3. **无需额外预训练**：直接利用已有的预训练视觉编码器

## 2. 相关工作

### 2.1 大型视觉-语言模型

视觉-语言模型经历了从两阶段训练到端到端训练的演进。早期工作如CLIP和ALIGN通过大规模视觉-语言对比学习建立了视觉-语言对齐。BLIP-2引入Q-Former进行跨模态特征转换。LLaVA使用线性投影连接视觉编码器和语言模型，实现了端到端的多模态理解。

然而，这些模型在组合视觉推理方面存在明显不足，缺乏对复杂视觉场景的结构化理解能力。

### 2.2 思维链提示

思维链（CoT）通过生成中间推理步骤显著提升了LLM的推理能力。自我一致性（Self-Consistency）进一步通过多路径采样和投票提升了推理可靠性。

将CoT思想迁移到视觉领域的工作包括：视觉思维链（Visual CoT）探索了视觉领域的链式推理；感知链（Perceive, Remember, Create）引入了记忆机制处理视觉任务。但这些方法在视觉表示层面缺乏显式的组合机制。

### 2.3 视觉组合性

视觉组合性研究关注模型理解和生成复杂视觉场景的能力。组合性泛化（compositional generalization）是核心挑战——模型能否通过已知组件的重新组合来理解新场景。

ShapeWorld等合成诊断数据集提供了量化评估组合推理能力的工具。这些数据集通过系统性地组合视觉元素（颜色、形状、关系等）来测试模型的组合能力。

## 3. 方法

### 3.1 整体框架

PoT的框架包含三个核心组件：

```
图像输入
  │
  ├─ CLIP ViT (像素级视觉特征) ──┐
  │                              │ 拼接 → 多粒度视觉编码器
  ├─ DINOv2 ViT (语义级特征) ────┘
  │         │
  ▼         ▼
原语提取器 (PE) ← LLM引导
  │
  ▼
原语聚合器 (PA) → 结构化区域图
  │
  ▼
答案解码器 (AD)
  │
  ▼
最终答案
```

### 3.2 双路视觉编码

视觉表示是VLM组合推理的基础。单一视觉编码器难以同时捕获语义级和像素级的视觉信息。因此，PoT采用双路视觉编码策略：

- **CLIP ViT**：提供像素级视觉特征，擅长图像-文本匹配任务
- **DINOv2 ViT**：提供语义级视觉特征，擅长细粒度分类和语义理解

两路特征通过拼接融合，形成多粒度视觉表示。

### 3.3 原语提取器（PE）

PE是PoT的核心创新模块。它接收多粒度视觉特征，输出离散的视觉原语序列。

**原语集合设计：**

| 原语类型 | 粒度级别 | 说明 | 示例 |
|----------|----------|------|------|
| P_cls | 最粗 | 对象类别 | "狗"、"猫"、"人" |
| P_attr | 细粒度 | 对象属性 | "红色"、"大"、"有斑点" |
| P_rel | 结构级 | 对象关系 | "在...上方"、"比...大" |
| P_part | 部位级 | 对象部位 | "狗的头部"、"猫的尾巴" |
| P_con | 抽象级 | 抽象概念 | "毛茸茸"、"优雅姿态" |

PE的实现：
- 使用轻量级Transformer架构
- 接收多粒度视觉特征和K个可学习的原语查询向量
- 输出K个视觉原语（K=16，平衡信息密度与计算效率）

### 3.4 原语聚合器（PA）

PA将PE输出的原语序列聚合成结构化表示：

1. **区域划分**：将图像划分为N×N个区域（论文使用7×7）
2. **区域-原语融合**：每个区域与所有原语进行交叉注意力融合
3. **区域图构建**：生成区域图，每个节点包含区域特征和关联的原语

区域图的数学表示：

```
G = (V, E, P)
其中：
- V = {v_1, v_2, ..., v_{N×N}} 是区域节点集
- E 是区域间的空间关系边
- P = {p_1, p_2, ..., p_K} 是原语集合
```

### 3.5 答案解码器（AD）

AD接收区域图表示和原始问题，通过LLM解码器输出最终答案：

- **输入**：区域图特征 + 问题文本
- **处理**：LLM对区域图进行推理，理解空间结构和原语关系
- **输出**：文本答案

### 3.6 训练策略

PoT采用两阶段训练：

**第一阶段：原语提取预训练**
- 目标：学习从图像中提取有效原语
- 数据：图像-原语配对数据集
- 冻结：视觉编码器和LLM
- 训练：PE和PA模块

**第二阶段：端到端指令微调**
- 目标：学习基于原语进行VQA
- 数据：VQA指令数据集
- 冻结：视觉编码器
- 训练：PE、PA和AD模块

## 4. 实验

### 4.1 诊断性实验：ShapeWorld

ShapeWorld是专门设计用于测试组合推理能力的合成数据集。每个样本由多个简单几何对象组合而成，通过系统性地改变对象数量、属性和关系来测试模型的组合泛化能力。

**组合准确率对比：**

| 模型 | 组合准确率 | 变化 |
|------|-----------|------|
| LLaVA | 36.2% | - |
| MiniGPT-4 | 38.5% | - |
| LLaVA-PoT（微调） | 57.3% | +21.1% |
| PoT（本文） | 62.1% | +25.9% |

**关键发现：** 微调后的LLaVA-PoT相比原始LLaVA提升了21个百分点，证明显式视觉原语分解对组合推理至关重要。PoT在此基础上进一步提升。

### 4.2 真实场景实验：GQA

GQA是包含多样化真实图像和复杂问题的VQA基准。

**GQA结果：**

| 模型 | 整体准确率 | 关系类 | 属性类 |
|------|-----------|--------|--------|
| LLaVA | 59.2% | 48.3% | 55.7% |
| MiniGPT-4 | 56.4% | 45.1% | 53.2% |
| LLaVA-PoT（微调） | 68.7% | 61.2% | 64.5% |
| PoT（本文） | 71.3% | 63.8% | 67.1% |

**关键发现：** PoT在关系推理和属性理解上分别提升约15和11个百分点，验证了方法迁移到真实场景的有效性。

### 4.3 消融实验

**视觉编码器消融（ShapeWorld）：**

| 配置 | 准确率 |
|------|--------|
| DINOv2 only | 48.2% |
| CLIP only | 44.6% |
| DINOv2 + CLIP（双路） | 62.1% |

**关键发现：** 双路视觉编码显著优于单路，验证了语义级和像素级特征互补的重要性。

**组件消融（ShapeWorld）：**

| 配置 | 准确率 |
|------|--------|
| 完整PoT | 62.1% |
| 无PA（去掉原语聚合器） | 54.3% |
| 无多粒度原语（仅类别原语） | 51.8% |

**关键发现：** PA组件贡献约8个百分点，多粒度原语设计贡献约10个百分点。

### 4.4 跨架构泛化

PoT框架对LLM骨干具有良好鲁棒性：

| LLM骨干 | ShapeWorld准确率 |
|---------|-----------------|
| Vicuna-7B | 59.7% |
| LLaMA-2-7B | 58.4% |

## 5. 分析与讨论

### 5.1 视觉原语的作用

视觉原语在组合推理中发挥三重作用：

1. **分解作用**：将复杂视觉场景分解为可管理的原语集合，降低推理难度
2. **结构化作用**：通过PA建立原语间的结构关系，支持空间和语义推理
3. **桥接作用**：作为视觉和语言模态之间的语义桥梁，促进跨模态理解

### 5.2 与文本CoT的对比

| 维度 | 文本CoT | PoT |
|------|---------|-----|
| 中间表示 | 文本推理步骤 | 视觉原语 |
| 可验证性 | 文本逻辑可验证 | 视觉原语可通过图像区域验证 |
| 分解粒度 | 逻辑步骤 | 视觉属性/关系/部位 |
| 多模态 | 仅文本 | 视觉+语言 |

### 5.3 局限性与未来工作

1. **原语集合有限**：当前5类原语可能无法覆盖复杂场景（如动态场景、3D推理）
2. **计算开销**：PE/PA两步结构增加约30-40%推理延迟
3. **真实场景提升幅度**：真实数据集提升幅度低于合成诊断集

未来工作可探索：
- 自动学习原语集合而非手工定义
- 扩展到视频和3D场景
- 与其他高效化方法结合

---

## 6. 结论

本文揭示了现有VLM在组合视觉推理方面的系统性缺陷，并通过诊断实验量化了这一问题。提出了视觉原语思维（PoT）范式，通过将复杂视觉输入分解为离散的视觉原语，为VLM提供"视觉工作记忆"。实验验证了PoT在组合推理任务上的显著提升，表明视觉原语作为中间表示层的有效性。
