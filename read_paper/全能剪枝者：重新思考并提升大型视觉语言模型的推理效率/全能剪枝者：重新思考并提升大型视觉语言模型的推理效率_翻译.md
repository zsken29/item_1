# 全能剪枝者：重新思考并提升大型视觉语言模型的推理效率

## Omni-Pruner: Unified Pruning for Large Vision-Language Models

**作者**: Yike Guo, Xin Wang, Wenwu Wang, Yu Li, Jin Liu, Hongxia Li
**机构**: 多所知名大学联合研究
**发表**: CVPR 2024
**arXiv**: 2403.10600

---

## 摘要

大型视觉语言模型（VLM）在视觉理解和语言生成方面展现出强大能力，但其推理效率仍然是部署的主要瓶颈。现有研究通常从单一维度优化VLM效率，如仅进行视觉Token剪枝或仅进行权重剪枝。然而，VLM的效率瓶颈存在于多个层面：视觉编码阶段的Token冗余、注意力机制的冗余计算、以及FFN层的参数冗余。单独优化某一维度无法充分利用所有优化机会。为解决这一问题，我们提出了**Omni-Pruner**，一个统一的多维度剪枝框架。Omni-Pruner的核心创新包括：（1）系统性地分析VLM各计算组件的效率瓶颈，建立完整的效率优化视图；（2）提出跨维度的联合剪枝策略，同时优化视觉Token、注意力头和FFN神经元；（3）设计帕累托最优搜索算法，在精度和效率之间找到最佳平衡点。实验表明，Omni-Pruner在LLaVA-1.5、InstructBLIP等模型上实现了5倍以上的整体加速，同时保持超过90%的原始性能。

## 1. 引言

### 1.1 研究背景

大型视觉语言模型（VLM）的架构通常包含三个主要组件：

```
┌─────────────────────────────────────────────────────────────────┐
│                    VLM架构示意图                                  │
│                                                                     │
│  输入图像                                                            │
│      ↓                                                              │
│  ┌─────────────────────────────────────────────────────────┐       │
│  │                 视觉编码器 (ViT)                         │       │
│  │  Layer 1-24: 产生576个视觉Token → 效率瓶颈1: Token冗余    │       │
│  └─────────────────────────────────────────────────────────┘       │
│      ↓                                                              │
│  ┌─────────────────────────────────────────────────────────┐       │
│  │                 连接层 (Projection)                       │       │
│  └─────────────────────────────────────────────────────────┘       │
│      ↓                                                              │
│  ┌─────────────────────────────────────────────────────────┐       │
│  │                 大语言模型 (LLM)                          │       │
│  │  Layer 1-32:                                             │       │
│  │  - 注意力头 (H=32): 冗余头 → 效率瓶颈2                   │       │
│  │  - FFN层: 冗余神经元 → 效率瓶颈3                         │       │
│  └─────────────────────────────────────────────────────────┘       │
│      ↓                                                              │
│  输出响应                                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 效率瓶颈的多元性

#### 瓶颈1：视觉Token冗余

CLIP ViT-L/14编码器产生576个视觉Token，其中大部分是冗余的：

| Token类型 | 比例 | 信息量 |
|----------|------|-------|
| 关键Token | 15% | 高 |
| 辅助Token | 25% | 中 |
| 冗余Token | 60% | 低 |

#### 瓶颈2：注意力头冗余

Vicuna-7B使用32个注意力头，部分头贡献较小：

```python
# 注意力头重要性分析
head_importance_scores = [
    0.95, 0.92, 0.89, 0.88, 0.87,  # 前5个头重要性高
    0.75, 0.72, 0.68, 0.65, 0.62,  # 中间5个头重要性中等
    0.45, 0.42, 0.38, 0.35, 0.32,  # 后5个头重要性低
    ...  # 总共32个头
]
```

#### 瓶颈3：FFN神经元冗余

FFN层通常包含大量冗余神经元：

- 隐藏维度：通常为4×模型维度
- 实际有效神经元：约60-70%
- 冗余神经元：约30-40%

### 1.3 现有方法的局限性

| 方法 | 优化维度 | 局限性 |
|------|---------|-------|
| FastV | 视觉Token | 仅优化视觉阶段 |
| LLM-Pruner | 权重剪枝 | 仅优化LLM |
| ATPM | Token剪枝 | 仅优化交互阶段 |
| SparseVLM | Token稀疏化 | 仅单一维度 |

**问题**：单一维度优化无法充分利用所有效率提升机会。

### 1.4 本文贡献

1. **系统分析**：全面分析VLM各阶段的效率瓶颈，建立完整的效率优化视图。

2. **Omni-Pruner框架**：提出统一的多维度剪枝框架，同时优化视觉Token、注意力头和FFN神经元。

3. **联合优化算法**：设计跨维度的联合优化算法，考虑不同剪枝维度之间的相互影响。

4. **帕累托最优搜索**：在精度-效率权衡曲线上找到帕累托最优解。

5. **实验验证**：在多个VLM和数据集上验证Omni-Pruner的有效性。

## 3. Omni-Pruner方法

### 3.1 问题定义

给定原始VLM $f(\cdot; \theta)$，其中 $\theta$ 包含视觉编码器参数 $\theta_V$、连接层参数 $\theta_P$ 和LLM参数 $\theta_L$。

**目标**：找到剪枝后的参数 $\theta'$，使得：

$$\min_{\theta'} \mathcal{L}_{task}(f(X; \theta'), Y) \quad \text{s.t.} \quad E(\theta') \geq E_{target}$$

其中 $E(\cdot)$ 是效率指标（加速比）。

### 3.2 多维度效率分析

#### 3.2.1 视觉阶段复杂度

$$T_{visual} = O(n_t \cdot d_v + n_t^2 \cdot d_v)$$

其中 $n_t$ 是Token数量，$d_v$ 是视觉特征维度。

#### 3.2.2 语言阶段复杂度

$$T_{language} = O(L \cdot (n_t + n_l)^2 \cdot d_l)$$

其中 $L$ 是LLM层数，$n_l$ 是文本Token数量，$d_l$ 是语言特征维度。

#### 3.2.3 各维度贡献分析

```
┌─────────────────────────────────────────────────────────────────┐
│  效率瓶颈贡献分析                                                  │
│                                                                     │
│  视觉Token:  ████████████████████████████░░░░░░░░░░  45%          │
│                                                                     │
│  注意力头:  █████████████████░░░░░░░░░░░░░░░░░░░░░  30%          │
│                                                                     │
│  FFN神经元: ██████████████░░░░░░░░░░░░░░░░░░░░░░░░  25%          │
│                                                                     │
│  总计: 100%                                                        │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 多维剪枝策略

#### 3.3.1 视觉Token剪枝

基于重要性分数选择保留的Token：

$$S_{keep} = \{i \mid s_i \in \text{TopK}(s, K)\}$$

其中 $s_i$ 是第 $i$ 个Token的重要性分数，$K = N \times (1 - r_t)$ 是保留的Token数量。

**重要性分数计算**：

$$s_i = \alpha \cdot \text{AttnScore}_i + \beta \cdot \|v_i\| + \gamma \cdot \text{Uniqueness}_i$$

#### 3.3.2 注意力头剪枝

对于每个注意力头 $h_k$，计算其重要性：

$$s_k^{head} = \frac{1}{T} \sum_{t=1}^{T} \frac{\partial \mathcal{L}}{\partial \text{Attn}(h_k^{(t)})}$$

保留重要头：

$$H_{keep} = \{k \mid s_k^{head} \in \text{TopK}(s^{head}, H \times (1 - r_h))\}$$

#### 3.3.3 FFN神经元剪枝

对于FFN层的中间神经元，计算重要性：

$$s_j^{neuron} = \sum_{l} \|W_{2,l}^{(j)}\| \cdot \|W_{1,l}^{(j)}\|$$

保留重要神经元：

$$N_{keep} = \{j \mid s_j^{neuron} \in \text{TopK}(s^{neuron}, M \times (1 - r_n))\}$$

### 3.4 联合优化算法

#### 3.4.1 优化目标

$$\min_{r_t, r_h, r_n} \mathcal{L}_{task} \quad \text{s.t.} \quad \sum_i \lambda_i \cdot r_i = R_{total}$$

其中 $\lambda_i$ 是各维度的效率权重，$R_{total}$ 是目标总压缩比。

#### 3.4.2 权重确定

$$\lambda_t : \lambda_h : \lambda_n = E_{visual} : E_{attn} : E_{ffn}$$

根据各维度对总效率的贡献比例确定。

#### 3.4.3 帕累托最优搜索

```
┌─────────────────────────────────────────────────────────────────┐
│  帕累托最优搜索                                                   │
│                                                                     │
│  输入: 压缩比范围 [r_min, r_max]，搜索步长 δ                        │
│  输出: 帕累托前沿点集合                                            │
│                                                                     │
│  1. 生成候选配置:                                                  │
│     for r_t in range(0, r_max, δ):                                │
│         for r_h in range(0, r_max, δ):                            │
│             r_n = R_total - λ_t*r_t - λ_h*r_h                     │
│             if r_n >= 0:                                          │
│                 candidates.append((r_t, r_h, r_n))                │
│                                                                     │
│  2. 评估每个配置:                                                  │
│     for config in candidates:                                     │
│         performance = evaluate(config)                            │
│         efficiency = compute_efficiency(config)                    │
│         pareto_points.add((efficiency, performance))              │
│                                                                     │
│  3. 提取帕累托前沿:                                                │
│     return pareto_front(pareto_points)                            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.5 训练与微调

#### 3.5.1 两阶段训练

**阶段1：重要性评估**

使用验证数据评估各组件的重要性分数。

**阶段2：剪枝+微调**

1. 根据重要性分数进行剪枝
2. 在指令微调数据上进行少量步数的微调
3. 恢复因剪枝导致的性能损失

#### 3.5.2 微调配置

- 学习率：2e-5（较预训练低）
- 批量大小：32
- 微调步数：500-1000
- 权重衰减：0.01

## 4. 实验

### 4.1 实验设置

#### 4.1.1 评估模型

| 模型 | 视觉编码器 | LLM | 总参数量 |

| COCO Captions | 图像描述 | 5k验证 |

### 4.2 主要结果

#### 4.2.1 VQAv2数据集

| 方法 | 压缩维度 | 总加速比 | VQAv2 | 性能保留 |
|------|---------|---------|-------|---------|
| 原始模型 | - | 1× | 80.0% | 100% |
| Token剪枝 | 视觉Token | 2× | 78.2% | 97.8% |
| 权重剪枝 | 注意力头+FFN | 2.5× | 77.5% | 96.9% |
| 单维剪枝（最佳） | 单一维度 | 2× | 78.0% | 97.5% |
| **Omni-Pruner** | **多维联合** | **5×** | **76.5%** | **95.6%** |

**关键发现**：
- Omni-Pruner在5倍加速下，性能仅下降4.4%
- 多维联合剪枝效果显著优于单维剪枝

#### 4.2.2 不同加速比的结果

| 加速比 | Token剪枝 | 注意力头剪枝 | FFN剪枝 | VQAv2 |

| 5× | 55% | 40% | 35% | 76.5% |

## 5. 结论

本文提出了Omni-Pruner，一个统一的多维度剪枝框架，用于全面提升大型视觉语言模型的推理效率。核心贡献包括：

1. **系统分析**：全面分析VLM各阶段的效率瓶颈，建立完整的效率优化视图。

2. **多维剪枝策略**：同时优化视觉Token、注意力头和FFN神经元三种维度。

3. **联合优化算法**：设计跨维度的联合优化算法，考虑不同剪枝维度之间的相互影响。

4. **帕累托最优搜索**：在精度-效率权衡曲线上找到帕累托最优解。

5. **实验验证**：在多个VLM和数据集上验证Omni-Pruner的有效性，实现5倍加速下90%+性能保留。

### 5.1 局限性

1. **训练成本**：联合优化需要更多训练时间。

2. **配置调优**：帕累托最优搜索需要大量实验确定最佳配置。

3. **任务泛化**：针对特定任务优化的剪枝配置可能在其他任务上表现不佳。

### 5.2 未来工作

1. **自适应多维剪枝**：根据输入内容动态调整各维度剪枝比例。

2. **与量化的结合**：将剪枝与量化技术结合实现更高压缩。

3. **硬件感知优化**：针对特定硬件特性优化剪枝模式。
