# 及时一针省九针：小型视觉语言模型是加速大型视觉语言模型的精准引导

## A Stitch in Time Saves Nine: Small Vision-Language Models as Precise Guidance for Accelerating Large VLMs

**作者**: Yike Guo, Xin Wang, Wenwu Wang, Yu Li, Jin Liu, Hongxia Li
**机构**: 多所知名大学联合研究
**发表**: ACL 2024
**arXiv**: 2406.04567

---

## 摘要

大型视觉语言模型（VLM）在各种视觉理解任务上取得了令人瞩目的成果，但其高昂的计算成本限制了实际部署。现有加速方法通常对所有输入采用统一的处理策略，忽视了输入复杂度的显著差异。实际上，大多数输入（~80%）是"简单的"，可以直接快速处理，无需调用完整的计算资源；而只有少部分"复杂的"输入需要大型模型的全部能力。为解决这一问题，我们提出了 **SewNet**，一种利用小型VLM引导大型VLM加速的协同推理框架。SewNet的核心创新是利用小型VLM作为"精准引导"，在推理早期识别输入的复杂度并预测关键区域，从而允许大型VLM采用更激进的剪枝策略。具体而言，小型VLM首先处理输入，生成注意力引导图和重要性分数，这些信号随后传递给大型VLM，使其能够聚焦于关键区域并跳过简单区域。实验表明，SewNet可以在LLaVA-1.5等大型模型上实现3-5倍加速，同时保持超过97%的原始性能。更重要的是，SewNet是即插即用的，可以与任何现有的VLM和加速方法结合使用。

## 1. 引言

### 1.1 研究背景

大型视觉语言模型（VLM）如GPT-4V、LLaVA和InstructBLIP展现了强大的视觉理解能力。然而，这些模型的计算成本令人望而却步：

| 模型 | 参数量 | 单次推理延迟 | 成本/次 |
|------|-------|------------|---------|
| GPT-4V | ~1T | ~5-10秒 | $0.03 |
| LLaVA-1.5-13B | 13B | ~800ms | 免费(本地) |
| LLaVA-1.5-7B | 7B | ~500ms | 免费(本地) |
| TinyLLaVA-3B | 3B | ~200ms | 免费(本地) |

### 1.2 输入复杂度的多样性

```
┌─────────────────────────────────────────────────────────────────┐
│  输入复杂度分布                                                    │
│                                                                     │
│  简单输入 (~80%):                                                  │
│  - "图中有几只猫？" → 快速回答"1只"                               │
│  - "这是什么颜色？" → 快速回答"红色"                               │
│  - "有人吗？" → "有"                                              │
│                                                                     │
│  复杂输入 (~20%):                                                  │
│  - "解释图中物理现象的工作原理" → 需要详细分析                     │
│  - "比较两幅图的异同" → 需要跨图像推理                             │
│  - "如果把A换成B会怎样" → 需要假设推理                            │
│                                                                     │
│  关键发现：80%的简单输入不需要完整的大型模型计算                    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 现有方法的局限性

#### （1）统一剪枝策略

现有方法对所有输入使用相同的剪枝比例，忽视了输入复杂度的差异：

```
┌─────────────────────────────────────────────────────────────────┐
│  统一剪枝策略                                                      │
│                                                                     │
│  简单输入 ──→ 统一剪枝50% ──→ 性能: 78% (过度剪枝)                │
│  复杂输入 ──→ 统一剪枝50% ──→ 性能: 65% (剪枝不足)                │
│                                                                     │
│  问题：无法适应输入多样性                                           │
└─────────────────────────────────────────────────────────────────┘
```

#### （2）静态剪枝阈值

固定阈值无法根据输入动态调整。

#### （3）缺乏引导机制

大型VLM无法利用小型模型的先验知识。

### 1.4 核心观察

**小型VLM可以作为有效的"先验"**：

1. **复杂度识别**：小型VLM可以准确判断输入的复杂度
2. **关键区域定位**：可以识别需要细粒度处理的关键区域
3. **计算节省**：简单输入可以在早期分流，节省大型模型计算

### 1.5 本文贡献

1. **问题洞察**：指出输入复杂度的多样性被忽视，提出"差异化加速"的理念。

2. **SewNet框架**：提出小型VLM引导大型VLM加速的协同推理框架。

3. **精准引导机制**：设计注意力引导和重要性预测两种引导信号。

4. **即插即用设计**：SewNet可以与任何现有VLM和加速方法结合使用。

5. **实验验证**：在多个VLM和数据集上验证SewNet的有效性。

## 3. SewNet方法

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    SewNet整体架构                                │
│                                                                     │
│  输入图像                                                            │
│      ↓                                                              │
│  ┌───────────────────────────────────────────────────────────┐     │
│  │                    小型VLM (TinyLLaVA)                      │     │
│  │  - 参数量: 3B                                               │     │
│  │  - 输出: 复杂度分数 + 注意力引导 + 重要性分数                 │     │
│  └───────────────────────────────────────────────────────────┘     │
│      ↓                                                              │
│  ┌───────────────────────────────────────────────────────────┐     │
│  │                    引导信号生成                            │     │
│  │  - G_attn: 注意力引导图                                     │     │
│  │  - s_imp: Token重要性分数                                   │     │
│  │  - c: 复杂度分数                                           │     │
│  └───────────────────────────────────────────────────────────┘     │
│      ↓                                                              │
│  ┌───────────────────────────────────────────────────────────┐     │
│  │                    大型VLM (LLaVA-1.5)                      │     │
│  │  - 参数量: 7B/13B                                           │     │
│  │  - 输入: 原始图像 + 引导信号                                 │     │
│  │  - 剪枝: 根据引导采用自适应策略                              │     │
│  └───────────────────────────────────────────────────────────┘     │
│      ↓                                                              │
│  输出响应                                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 复杂度识别

#### 3.2.1 复杂度分数

小型VLM输出复杂度分数 $c \in [0, 1]$：

$$c = \sigma(W_c \cdot h_{small} + b_c)$$

其中 $h_{small}$ 是小型VLM的最终隐藏状态。

#### 3.2.2 复杂度分类

| 复杂度等级 | 分数范围 | 处理策略 |
|-----------|---------|---------|
| 简单 | $c < 0.3$ | 极激进剪枝 (70-80%) |
| 中等 | $0.3 \leq c < 0.7$ | 中等剪枝 (40-50%) |
| 复杂 | $c \geq 0.7$ | 保守剪枝 (20-30%) |

### 3.3 引导信号生成

#### 3.3.1 注意力引导图

小型VLM的注意力图作为空间引导：

$$G_{attn} = \text{Attention}(h_{small}, I)$$

其中 $I$ 是输入图像。

```
┌─────────────────────────────────────────────────────────────────┐
│  注意力引导图示例                                                  │
│                                                                     │
│  输入图像: 包含人物和背景的复杂场景                                  │
│                                                                     │
│  小型VLM注意力:                                                     │
│  [淡色表示低关注，淡红色表示中等关注，深红色表示高度关注]             │
│                                                                     │
│  引导结果:                                                          │
│  - 人物区域: 高度关注 → 保留                                       │
│  - 背景区域: 低关注 → 可剪枝                                       │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.3.2 重要性分数

预测每个Token的重要性分数：

$$s_i = \text{MLP}([h_{small}; v_i])$$

其中 $v_i$ 是第 $i$ 个视觉Token，$[;]$ 表示拼接。

### 3.4 自适应剪枝

#### 3.4.1 剪枝策略

根据复杂度分数 $c$ 自适应调整剪枝比例：

$$r_{final} = f(c, r_{base})$$

| 复杂度 | 基础剪枝 $r_{base}$ | 自适应调整 | 最终剪枝 $r_{final}$ |
|--------|---------------------|-----------|---------------------|
| 简单 | 50% | +20% | 70% |
| 中等 | 50% | 0% | 50% |
| 复杂 | 50% | -20% | 30% |

#### 3.4.2 引导感知Token选择

保留Token时优先考虑引导信号：

$$T_{keep} = \text{TopK}_{i \in S_{high}} (s_i + \lambda \cdot G_{attn}^{(i)})$$

其中 $S_{high}$ 是高注意力区域对应的Token集合。

### 3.5 两阶段推理流程

```
┌─────────────────────────────────────────────────────────────────┐
│  SewNet推理流程                                                   │
│                                                                     │
│  输入: 图像 I，复杂度阈值 τ                                        │
│                                                                     │
│  阶段1: 小型VLM处理                                                │
│  ──────────────────────                                           │
│  1. I_small = resize(I, target_size)                              │
│  2. h_small = SmallVLM(I_small)                                  │
│  3. c = sigmoid(W_c · h_small + b_c)  # 复杂度分数               │
│  4. G_attn = Attention(h_small, I)  # 注意力引导                  │
│  5. s_imp = MLP([h_small; V])  # 重要性分数                      │
│                                                                     │
│  if c < τ:                                                        │
│      # 简单输入，直接返回                                          │
│      return generate_response(h_small)                             │
│                                                                     │
│  阶段2: 大型VLM处理                                                │
│  ──────────────────────                                           │
│  6. r = adaptive_pruning_ratio(c, r_base)  # 自适应剪枝比例       │
│  7. T_keep = guided_token_selection(G_attn, s_imp, r)  # 引导选择 │
│  8. V_pruned = select(V, T_keep)                                  │
│  9. h_large = LargeVLM(V_pruned, guidance=G_attn)                 │
│  10. return generate_response(h_large)                            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.6 与现有方法的兼容性

SewNet是即插即用的，可以与以下方法结合：

| 方法 | 结合方式 | 预期效果 |
|-----|---------|---------|
| FastV | 在引导后应用FastV剪枝 | 双重加速 |
| LLM-Pruner | 使用引导信号辅助权重剪枝 | 性能恢复 |
| 量化 | 引导+量化 | 效率倍增 |
| 早退 | 引导决定退出时机 | 动态深度 |

## 4. 实验

### 4.1 实验设置

#### 4.1.1 模型配置

| 模型 | 参数量 | 视觉编码器 | LLM类型 |
|------|-------|-----------|---------|
| SmallVLM (引导器) | 3B | CLIP ViT-S | TinyLLaMA |
| LargeVLM (被加速) | 7B | CLIP ViT-L | Vicuna-7B |
| LargeVLM (被加速) | 13B | CLIP ViT-L | Vicuna-13B |

#### 4.1.2 基准数据集

| 数据集 | 任务类型 | 样本数 | 复杂度分布 |
|-------|---------|-------|-----------|
| VQAv2 | 视觉问答 | 83k验证 | 60%简单 |
| GQA | 视觉推理 | 12k验证 | 40%简单 |
| VQA-CE | 挑战性VQA | 125k测试 | 20%简单 |
| POPE | 幻觉检测 | 3k验证 | 多样 |

### 4.2 主要结果

#### 4.2.1 VQAv2数据集

| 方法 | VQAv2 | 加速比 | 性能保留 |
|------|-------|-------|---------|
| 原始 (LLaVA-7B) | 80.0% | 1× | 100% |
| 统一剪枝 50% | 78.2% | 2× | 97.8% |
| FastV | 78.5% | 2.2× | 98.1% |
| LLaVA-PruMerge | 78.8% | 2.5× | 98.5% |
| **SewNet (Ours)** | **78.8%** | **3×** | **98.5%** |

**关键发现**：SewNet在相同性能下实现更高加速比。

#### 4.2.2 不同复杂度输入

| 输入类型 | 占比 | 方法 | 准确率 | 加速比 |
|---------|------|------|-------|--------|
| 简单 | 60% | SewNet | 99.5% | 5× |
| 简单 | 60% | 统一剪枝 | 98.5% | 2× |
| 复杂 | 40% | SewNet | 76.5% | 2× |
| 复杂 | 40% | 统一剪枝 | 74.2% | 2× |

**结论**：SewNet在简单输入上实现更高加速，在复杂输入上保持性能。

#### 4.2.3 GQA数据集

| 方法 | 准确率 | 加速比 |
|------|--------|-------|
| 原始 | 62.0% | 1× |
| 统一剪枝 | 60.2% | 2× |
| FastV | 60.5% | 2.2× |
| **SewNet** | **60.8%** | **2.8×** |

#### 4.2.4 VQA-CE数据集

| 方法 | 准确率 | 加速比 |
|------|--------|-------|
| 原始 | 48.2% | 1× |
| 统一剪枝 | 45.8% | 2× |
| **SewNet** | **46.8%** | **2.5×** |

### 4.3 消融实验

#### 4.3.1 引导信号消融

| 引导类型 | VQAv2 | 加速比 |
|---------|-------|--------|

## 5. 结论

本文提出了SewNet，一种利用小型VLM引导大型VLM加速的协同推理框架。核心贡献包括：

1. **问题洞察**：指出输入复杂度的多样性被忽视，提出差异化加速的理念。

2. **SewNet框架**：设计小型VLM引导大型VLM的自适应加速方案。

3. **精准引导机制**：提出注意力引导和重要性预测两种引导信号。

4. **即插即用设计**：SewNet可以与任何现有VLM和加速方法结合使用。

5. **实验验证**：在多个VLM和数据集上验证SewNet的有效性，实现3倍加速下97%+性能保留。

### 5.1 局限性

1. **小型模型质量**：引导质量依赖于小型VLM的能力。

2. **内存占用**：需要同时运行两个模型，内存占用增加。

3. **推理调度**：需要额外的逻辑处理模型切换。

### 5.2 未来工作

1. **更小的引导模型**：探索1B或更小的引导模型。

2. **多级级联**：扩展到多级级联（小小→小→大）。

3. **动态引导策略**：根据中间结果动态调整引导。
