# 当Token说得太多：多模态长上下文Token压缩综述

**When LLMs Talk Too Much: A Survey on Multimodal Long-Context Token Compression**

**原文链接：** https://arxiv.org/abs/XXXX.XXXXX

---

## 摘要

随着多模态大语言模型（MLLMs）的快速发展，处理包含图像、视频和音频的长上下文输入已成为核心能力。然而，长上下文带来的巨大计算和内存开销严重限制了模型的实际部署和推理效率。Token压缩作为一种有效的效率优化手段，通过减少冗余的视觉Token来加速多模态推理，同时尽量保持模型性能。

本综述系统梳理了多模态长上下文Token压缩的研究进展，从压缩粒度、压缩策略、训练范式和应用场景四个维度对现有方法进行了全面分类和分析。我们深入探讨了不同压缩方法的优缺点，并展望了该领域的未来发展方向。

**核心贡献：**
- 首个系统梳理多模态Token压缩方法的综述
- 提出四维度分类框架：压缩粒度、压缩策略、训练范式、应用场景
- 覆盖图像、视频、音频三种模态的压缩技术
- 为该领域的未来研究提供系统性指导

**关键词：** 多模态大语言模型、长上下文、Token压缩、视觉Token剪枝、效率优化

## 1. 引言

### 1.1 研究背景

多模态大语言模型的崛起代表了人工智能发展的重要里程碑。现代MLLMs能够处理图像、视频、音频和文本的任意组合，展现出强大的多模态理解和生成能力。然而，这种强大的能力伴随着显著的计算和内存开销：

**挑战一：视觉Token数量庞大**
- 高分辨率图像被编码为成千上万个Token
- 视频包含时间维度，Token数量随帧数线性增长
- 音频信号经过采样和编码后产生大量Token

**挑战二：计算复杂度高**
- 自注意力机制的复杂度为O(n²)，其中n为Token序列长度
- 长序列导致注意力计算成为主要瓶颈
- KV缓存的内存占用随序列长度线性增长

**挑战三：部署成本高**
- 长上下文模型需要大量GPU显存
- 推理延迟严重影响用户体验
- 边缘设备部署几乎不可能

### 1.2 Token压缩的动机

Token压缩的核心理念是：并非所有视觉Token对最终任务同等重要。通过识别和保留关键Token，同时去除冗余Token，可以在几乎不损失性能的情况下显著加速推理。

**为什么需要Token压缩？**
- 模型接收的视觉信息远超人眼短期记忆容量
- 大多数视觉Token对具体任务贡献有限
- 冗余的Token增加计算负担但不提供相应价值
- 知识压缩是实现高效推理的有效手段

### 1.3 综述范围与结构

本综述涵盖以下内容：
- **模态范围**：图像、视频、音频三种模态
- **压缩方法**：从粗粒度到细粒度的各种压缩策略
- **训练范式**：无需训练的方法、稀疏训练方法、端到端训练方法
- **应用场景**：视觉问答、视觉推理、视频理解、语音识别等

## 2. 预备知识

### 2.1 多模态大语言模型架构

**典型架构组件：**

1. **视觉编码器（Visual Encoder）**
   - 负责将图像/视频/音频编码为Token序列
   - 常用架构：ViT、CLIP、EVA-CLIP等
   - 输出：视觉Token序列

2. **投影层（Projection Layer）**
   - 将视觉Token映射到语言模型的空间
   - 常用方法：MLP、Q-Former等
   - 作用：模态对齐

3. **大语言模型（LLM）**
   - 核心推理和生成引擎
   - 接收多模态Token序列
   - 输出文本响应

### 2.2 视觉Token的产生过程

**图像Token化：**
- 图像被分割成patches（如16x16像素）
- 每个patch被线性投影为Token
- 224x224图像 → 196个Token（14x14 grid）
- 高分辨率图像（如1024x1024）→ 4096+ Tokens

**视频Token化：**
- 视频被分解为帧序列
- 每帧独立进行图像Token化
- 时间维度增加Token数量：帧数 × 每帧Token数
- 30帧视频 → 30 × 196 = 5,880 Tokens

**音频Token化：**
- 音频被分割成短时窗口
- 每个窗口被编码为Token序列
- Whisper等模型产生大量音频Token

### 2.3 注意力机制与计算复杂度

**自注意力的复杂度：**
```
Complexity = O(n² × d)
```
其中n为序列长度，d为隐藏维度。

**实际影响：**
- n=1000时，注意力计算已相当耗时
- n=10000时，注意力计算成为主要瓶颈
- KV缓存大小随n线性增长：2 × n × d × h（h为层数）

## 3. 压缩粒度分类

### 3.1 粗粒度压缩（Coarse-grained Compression）

**定义：** 在Token序列的早期阶段进行大幅压缩，一次性去除大量Token。

**代表性方法：**

1. **全局压缩**
   - 将整个视觉序列压缩为固定数量的Token
   - 保留全局结构信息
   - 丢失细节信息

2. **空间粗粒度压缩**
   - 在空间维度上进行下采样
   - 类似于图像金字塔
   - 减少空间分辨率

3. **时间粗粒度压缩（主要用于视频）**
   - 减少时间采样率
   - 跳帧处理
   - 降低时间分辨率

**优点：**
- 压缩效率高
- 计算开销小
- 实现简单

**缺点：**
- 可能丢失关键细节
- 难以恢复已压缩的信息
- 对压缩比例敏感

### 3.2 细粒度压缩（Fine-grained Compression）

**定义：** 在Token级别进行选择性压缩，保留重要Token，移除不重要Token。

**代表性方法：**

1. **Token重要性评分**
   - 为每个Token计算重要性分数
   - 基于分数进行选择性保留
   - 保留top-k%的重要Token

2. **Token聚类**
   - 将相似Token聚类
   - 每个簇保留一个代表性Token
   - 减少冗余

3. **Token合并**
   - 合并语义相似的Token
   - 生成更紧凑的表示
   - 保持语义完整性

**优点：**
- 保留关键信息
- 自适应压缩
- 性能损失小

**缺点：**
- 需要额外的评分模块
- 增加推理开销
- 重要性评估可能不准确

### 3.3 混合粒度压缩（Hybrid Compression）

**定义：** 结合粗粒度和细粒度压缩的优势，采用多阶段压缩策略。

**典型流程：**
1. 粗粒度阶段：快速减少Token数量
2. 细粒度阶段：选择性保留关键Token
3. 混合阶段：结合全局和局部信息

**优点：**
- 平衡效率和精度
- 灵活适应不同场景
- 性能最优

## 4. 压缩策略分类

### 4.1 基于重要性评分的压缩

**核心思想：** 学习或设计Token的重要性评分函数，保留高重要性Token。

**方法分类：**

1. **可学习的评分函数**
   - 训练专门的评分网络
   - 端到端优化评分准确性
   - 需要标注数据

2. **基于注意力的评分**
   - 利用注意力权重作为重要性指标
   - 高注意力 → 高重要性
   - 无需额外训练

3. **基于重建的评分**
   - 评估Token对重建原始信息的贡献
   - 高重建误差的Token更重要
   - 需要重建网络

**关键问题：**
- 如何定义"重要性"？
- 重要性是否与任务相关？
- 如何平衡重要性和多样性？

### 4.2 基于聚类的压缩

**核心思想：** 将语义相似的Token聚类，用簇中心或簇代表表示整个簇。

**方法分类：**

1. **空间聚类**
   - 在特征空间中进行聚类
   - 保留空间结构
   - 适合图像

2. **语义聚类**
   - 基于语义相似性聚类
   - 保留语义多样性
   - 适合复杂场景

3. **层次聚类**
   - 多层次聚类结构
   - 支持变长压缩
   - 适应不同需求

**代表方法：**
- K-means聚类压缩
- 层次聚类压缩
- 基于原型的压缩

### 4.3 基于注意力的压缩

**核心思想：** 利用注意力机制识别关键Token和Token间关系。

**方法分类：**

1. **注意力权重压缩**
   - 保留高注意力权重的Token
   - 基于attention map选择
   - 实现简单

2. **注意力稀疏化**
   - 引入稀疏注意力模式
   - 减少注意力计算量
   - 近似全注意力

3. **跨模态注意力引导**
   - 利用文本注意力指导视觉Token选择
   - 选择与文本高度相关的视觉Token
   - 提高任务相关 性

### 4.4 可学习的压缩策略

**核心思想：** 通过神经网络学习最优的压缩策略。

**方法分类：**

1. **端到端学习**
   - 将压缩模块整合到模型中
   - 端到端优化压缩决策
   - 性能最优但训练复杂

2. **强化学习压缩**
   - 将压缩问题建模为序列决策
   - 最大化任务性能
   - 探索高效压缩策略

3. **知识蒸馏压缩**
   - 从大模型蒸馏压缩知识
   - 训练轻量压缩模型
   - 继承大模型能力

## 5. 训练范式分类

### 5.1 无需训练的压缩（Training-free）

**核心思想：** 不需要额外训练，直接基于预训练模型设计压缩策略。

**代表性方法：**

1. **基于统计的方法**
   - 利用Token的统计特征（频率、方差等）
   - 识别冗余Token
   - 无需训练

2. **基于相似度的方法**
   - 计算Token间的相似度
   - 移除高度相似的Token
   - 保留多样性

3. **基于启发式的方法**
   - 设计人工规则指导压缩
   - 简单高效
   - 可解释性强

**优点：**
- 无需额外训练
- 通用性强
- 部署简单

**缺点：**
- 性能有限
- 难以针对特定任务优化
- 可能不是最优策略

### 5.2 稀疏训练（Sparse Training）

**核心思想：** 在训练过程中引入稀疏性，训练时只更新重要的参数和Token。

**代表性方法：**

1. **Token稀疏化训练**
   - 训练时随机稀疏化Token
   - 学习哪些Token值得保留
   - 减少训练和推理成本

2. **梯度稀疏化**
   - 只对重要Token计算梯度
   - 减少反向传播开销
   - 加速训练

3. **动态稀疏化**
   - 在不同训练阶段稀疏化不同Token
   - 探索更大的Token空间
   - 发现更优压缩策略

**优点：**
- 训练效率高
- 可发现新压缩策略
- 平衡性能和效率

**缺点：**
- 训练复杂
- 需要仔细的超参数调整
- 稀疏度选择困难

### 5.3 端到端训练（End-to-end Training）

**核心思想：** 将压缩模块作为模型的一部分，进行端到端优化。

**代表性方法：**

1. **联合优化**
   - 压缩模块和主模型联合训练
   - 端到端梯度传播
   - 性能最优

2. **两阶段训练**
   - 阶段一：训练压缩模块
   - 阶段二：微调整个系统
   - 训练稳定

3. **持续训练**
   - 在压缩后继续训练
   - 补偿压缩带来的性能损失
   - 保持模型能力

**优点：**
- 性能最优
- 针对性强
- 可充分利用数据

**缺点：**
- 训练成本高
- 需调整压缩模块
- 可能过拟合特定任务

## 6. 应用场景分析

### 6.1 视觉问答（Visual Question Answering）

**场景特点：**
- 输入：图像 + 文本问题
- 输出：文本答案
- 特点：需要理解图像内容和问题意图

**Token压缩的作用：**
- 减少与问题无关的视觉Token
- 保留回答问题所需的关键视觉信息
- 加速问答推理

**挑战：**
- 问题相关性的准确判断
- 保持空间理解能力
- 处理多样化的视觉内容

### 6.2 视觉推理（Visual Reasoning）

**场景特点：**
- 输入：复杂图像或多图关系
- 输出：推理结果
- 特点：需要多步推理和逻辑分析

**Token压缩的作用：**
- 保留推理所需的关键视觉线索
- 减少干扰信息
- 支持多步推理链

**挑战：**
- 保持细粒度的空间关系理解
- 处理多图像之间的关联
- 支持复杂的逻辑推理

### 6.3 视频理解（Video Understanding）

**场景特点：**
- 输入：视频序列（多帧图像 + 音频）
- 输出：视频内容理解
- 特点：时间维度增加复杂性

**Token压缩的作用：**
- 减少帧级Token数量
- 保留关键帧和关键视觉事件
- 平衡时间和空间信息

**挑战：**
- 时间信息的保留
- 关键动作的识别
- 音频和视频的联合压缩

### 6.4 文档理解（Document Understanding）

**场景特点：**
- 输入：文档图像（含文本、表格、图表）
- 输出：文档内容理解
- 特点：需要理解丰富的视觉布局

**Token压缩的作用：**
- 压缩背景和装饰性元素
- 保留关键文本和结构信息
- 支持复杂布局理解

**挑战：**
- 保持文本识别能力
- 理解文档结构
- 处理多栏和复杂布局

## 7. 评估与基准

### 7.1 评估指标

**效率指标：**
- **Token减少比例**：压缩后Token减少的百分比
- **加速比**：压缩后的推理速度提升
- **内存节省**：KV缓存和激活内存的减少

**性能指标：**
- **任务准确率**：在下游任务上的准确率
- **性能保持率**：压缩后性能与原始性能的比值
- **细粒度性能**：在不同难度问题上的表现

**权衡指标：**
- **效率-性能权衡曲线**：不同压缩比例下的性能
- **帕累托最优**：效率-性能的帕累托前沿

### 7.2 评估基准

**通用多模态基准：**
- VQAv2：视觉问答基准
- GQA：视觉推理基准
- VCR：视觉Commonsense推理

**特定任务基准：**
- ChartQA：图表理解
- DocVQA：文档理解
- TVQA：视频问答

**效率评估基准：**
- 推理延迟测试
- 内存占用测试
- 吞吐量测试

### 7.3 常用数据集

**图像数据集：**
- COCO：通用目标检测和图像描述
- Visual Genome：场景图和视觉关系
- LAION：大规模图文对

**视频数据集：**
- ActivityNet：人类活动视频
- YouTube-8M：大规模视频理解
- HowTo100M：教学视频

**音频数据集：**
- AudioSet：音频事件分类
- LibriSpeech：语音识别
- VoxCeleb：说话人识别

## 8. 挑战与未来方向

### 8.1 当前挑战

**挑战一：压缩与性能的权衡**
- 高压缩率导致显著性能下降
- 如何在极端压缩下保持关键能力
- 寻找最优的效率-性能平衡点

**挑战二：通用性与针对性的矛盾**
- 通用压缩方法难以适应所有任务
- 针对特定任务的压缩可能泛化性差
- 如何实现任务自适应的压缩

**挑战三：评估标准不统一**
- 不同论文使用不同的评估指标
- 难以公平比较不同方法
- 需要建立统一的评估标准

**挑战四：理论理解的不足**
- 为什么压缩有效缺乏理论解释
- 什么样的压缩策略是最优的
- 压缩的极限在哪里

### 8.2 未来方向

**方向一：自适应压缩**
- 根据输入内容自适应调整压缩比例
- 不同区域/时间采用不同压缩策略
- 动态平衡效率和精度

**方向二：跨模态联合压缩**
- 图像、视频、音频统一压缩框架
- 跨模态冗余识别和去除
- 模态间信息互补利用

**方向三：任务感知压缩**
- 根据下游任务调整压缩策略
- 针对特定任务优化压缩模块
- 任务相关的Token重要性评估

**方向四：高效压缩架构**
- 设计专门的高效视觉编码器
- 压缩友好的模型架构
- 硬件感知的压缩设计

**方向五：压缩与训练的协同**
- 压缩aware的训练方法
- 在训练阶段就考虑压缩因素
- 从源头设计可压缩的模型

## 9. 总结

本综述系统梳理了多模态长上下文Token压缩的研究进展。我们从压缩粒度、压缩策略、训练范式和应用场景四个维度对现有方法进行了分类和分析。

**主要发现：**
1. Token压缩是解决多模态长上下文计算瓶颈的有效方法
2. 不同压缩方法各有优缺点，适用于不同场景
3. 端到端学习的压缩方法通常能获得最佳性能
4. 压缩与性能的权衡是核心挑战

**关键洞察：**
- 重要性评分的准确性是细粒度压缩的关键
- 任务相关的压缩策略优于通用策略
- 自适应压缩是未来发展的重要方向
- 跨模态联合压缩具有巨大潜力

**研究建议：**
- 关注压缩方法的可解释性和可控性
- 建立统一的评估标准和基准
- 探索压缩与模型架构的协同设计
- 研究压缩对模型能力的长远影响
