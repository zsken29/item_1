## 二、核心问题与动机

### 2.1 背景：视觉Token剪枝的广泛研究

过去几年，众多工作从不同角度研究视觉Token剪枝：

| 剪枝策略 | 代表方法 | 基本原理 |
|----------|---------|---------|
| 注意力熵 |SparseLLM等 | 熵高的Token贡献更多信息，应保留 |
| L2范数 | LLMRazor等 | 范数大的Token表示更显著，应保留 |
| 语义聚类 | EVOPRUN等 | 同类Token保留代表，减少冗余 |
| 梯度敏感度 | ATS-Pruning | 梯度大的Token对损失影响大，应保留 |

这些方法在**整体准确率**上表现良好，但本文揭示了一个关键问题：**整体指标掩盖了局部失败**。

### 2.2 被忽视的关键现象：RVIS

**相关视觉信息偏移（Relevant Visual Information Shift, RVIS）**是本文提出的核心概念：

> **定义**：在回答特定问题时，原始Token集合中与答案相关的视觉Token，被剪枝掉的比例。

**直观理解**：

举例来说，"图像左边的物体是什么？"这个问题：
- 相关Token = 图像左侧的Token（可能占10-20%的Token）
- 无关Token = 图像右侧和背景的Token（占80-90%的Token）

若剪枝策略基于"语义显著性"保留Token，往往优先保留中心区域、显著物体，而**系统性地遗漏边缘区域**。对于"左侧"这类空间查询，问题必然失败。

### 2.3 为什么现有指标不够

以GQA数据集为例：

| 评估方式 | 准确率 |
|----------|--------|
| 整体准确率 | 72.3% |
| 物体识别子任务 | 85.6% |
| 空间关系子任务 | 54.1% |
| 计数子任务 | 61.2% |

**整体准确率72.3%看起来可以接受，但空间关系任务54.1%和计数任务61.2%在实际应用中可能是不可接受的。**

更关键的是，模型在错误预测时置信度仍然很高（~85%），说明**模型完全不知道自己答错了**。

## 三、方法详解

### 3.1 分析框架：四步走

本文的分析框架分为四个步骤：

**Step 1: Attention追踪**
- 在MLLM的每个Attention层中，提取Query对视觉Token的Attention权重
- 汇总各层权重，得到模型"关注"的视觉Token分布

**Step 2: 相关性标注**
- 使用GPT-4V对图像中每个Token（Patch）标注其与特定问题的相关性
- 三级标注：相关 / 中立 / 不相关

**Step 3: 偏移量化**
- 计算剪枝前后，相关Token被保留的比例
- $RVIS = 1 - \frac{|R(q,V) \cap V'|}{|R(q,V)|}$
- RVIS = 0.0 表示没有相关信息丢失；RVIS = 1.0 表示所有相关信息都被剪掉

**Step 4: 失败模式分析**
- 按问题类型聚合RVIS率
- 识别不同剪枝策略的脆弱点

### 3.2 重要性评分方法

各剪枝策略的重要性评分方式：

**注意力熵剪枝**：
$$s_j = -\sum_h \sum_{k} p_{h,j,k} \log p_{h,j,k}$$

Token $j$ 在各Attention head中作为Key时的分布熵。熵高 → Token与多个Query交互 → 信息量高 → 应保留。

**L2范数剪枝**：
$$s_j = \|v_j\|_2$$

Token embedding的L2范数。范数大 → 表示更显著 → 应保留。

**语义聚类剪枝**：
$$s_j = \text{distance}(v_j, \text{cluster\_center}(c_j))$$

Token与其所属语义聚类中心的距离。距离大 → 更边缘 → 先剪枝。

### 3.3 问题类型分类

将问题分为5类进行分析：

| 类型 | 描述 | 剪枝敏感性 |
|------|------|-----------|
| 物体识别 | 询问"图中有什么物体" | 低（剪枝后仍有显著物体信息） |
| 空间关系 | 询问"物体A在B的哪个方向" | 高（需要精确边界信息） |
| 计数 | 询问"图中有几个X" | 高（需要分散的全图信息） |
| 属性识别 | 询问"X是什么颜色/形状" | 中（需要物体级信息） |
| 动作识别 | 询问"正在做什么" | 中（需要主体+背景信息） |

## 四、实验发现详解

### 4.1 发现一：RVIS与剪枝率的关系

实验数据（以注意力熵剪枝为例）：

| 保留Token比例 | 空间关系RVIS率 | 计数RVIS率 | 物体识别RVIS率 |
|---------------|----------------|------------|----------------|
| 100% | 0% | 0% | 0% |
| 75% | 12% | 8% | 2% |
| 50% | 31% | 24% | 5% |
| 25% | 58% | 47% | 11% |

**结论**：
- 剪枝率超过50%时，空间关系和计数任务的RVIS急剧上升
- 物体识别任务对剪枝最鲁棒，因为显著物体通常被保留
- 所有任务在75%保留率时RVIS可控，但到50%时问题显著

### 4.2 发现二：不同剪枝策略的脆弱点

| 策略 | 空间关系 | 计数 | 物体识别 | 动作 |
|------|---------|------|---------|------|
| 注意力熵 | 最差 | 中 | 良好 | 中 |
| L2范数 | 中 | 最差 | 良好 | 中 |
| 语义聚类 | 良好 | 中 | 中 | 最差 |

**结论**：无"万能策略"。每种策略在某些任务上有效，在其他任务上有缺陷。

原因分析：
- **注意力熵**：优先保留"被多方关注"的Token，这些Token往往位于图像中心或显著物体上。边缘和边界Token被剪掉 → 空间关系失败率高
- **L2范数**：优先保留高范数Token，这通常对应纹理丰富的区域。分散的低纹理区域被剪掉 → 计数失败率高
- **语义聚类**：每个聚类只保留少数代表Token。语义多样性高的区域被过度压缩 → 动作识别失败率高

### 4.3 发现三：模型缺乏自感知能力

当RVIS发生时，模型的行为：

```
剪枝前：
  - 相关Token Attention总和：A_r = 0.72
  - 预测置信度：85.3%
  - 预测正确率：91.2%

剪枝后（50%保留，RVIS发生）：
  - 相关Token Attention总和：A_r' = 0.21（大幅下降）
  - 模型转而关注中立Token：A_n = 0.64（上升）
  - 预测置信度：83.7%（几乎没有变化！）
  - 预测正确率：54.1%（大幅下降）
```

**关键问题**：置信度在剪枝后几乎没有下降，说明模型完全意识不到自己错过了关键视觉信息。

这是一个严重的安全隐患：在实际应用中，用户无法判断模型的预测是基于"正确的视觉依据"还是"错误的相关信息"。

### 4.4 发现四：评估指标的掩盖效应

| 方法 | 整体Acc | 空间关系 | 计数 | 物体识别 |
|------|---------|---------|------|---------|
| 无剪枝 | 72.3% | 78.5% | 71.2% | 89.3% |
| 注意力熵剪枝(50%) | 69.8% | 54.1% | 61.2% | 85.6% |
| L2范数剪枝(50%) | 69.1% | 62.3% | 52.8% | 84.9% |

整体准确率的下降（72.3% → 69.8%）看起来温和，但空间关系从78.5%降到54.1%，下降了近25个百分点！

**建议**：未来的视觉Token剪枝工作应报告细粒度的任务级准确率。

## 五、改进方向与建议

### 5.1 任务感知剪枝策略

**空间关系任务**：
- 保留更多图像边缘区域（包含边界信息）
- 保留包含空间参考点的Token（如"桌子边缘"、"物体边界"）

**计数任务**：
- 保留均匀分布在图像各区域的Token
- 避免Token过度集中于某一块

### 5.2 相关性感知的剪枝

核心思想：在剪枝决策中引入"与当前问题的相关性"信息。

方法：
1. **问题引导的重要性评分**：在计算Token重要性时，不仅考虑Token的固有特性（熵、范数），还考虑Token与问题的语义关联
   $$s_j' = s_j \cdot \text{sim}(v_j, q)$$
   其中 $\text{sim}(v_j, q)$ 是Token与问题文本的语义相似度

2. **两阶段剪枝**：先用固有重要性剪枝到75%，再用问题相关性剪枝到50%

### 5.3 鲁棒性评估指标

建议未来评估包含以下指标：

| 指标 | 含义 |
|------|------|
| RVIS率 | 剪枝导致的相关Token丢失比例 |
| 任务级准确率 | 按问题类型分组的准确率 |
| 置信度校准 | 置信度与实际准确率的匹配程度 |
| 失败案例分析 | 抽样分析失败案例的视觉原因 |

## 七、实践启示

1. **不要只看整体准确率**：评估剪枝方法时，必须按问题类型分组报告准确率
2. **高剪枝率有风险**：75%保留率以上是安全的；50%保留率以上需谨慎；25%保留率在空间关系任务上不可用
3. **空间关系任务**：应避免基于注意力熵的剪枝策略（优先剪边缘）
4. **计数任务**：应避免基于L2范数的剪枝策略（优先剪低纹理区域）
5. **置信度不可信**：剪枝后模型的置信度不能反映预测的可靠性
