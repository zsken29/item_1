**发表**: ACL 2024 (Findings)

## 摘要

视觉语言模型（VLM）的压缩对于实现高效部署至关重要。现有方法通常从单一维度进行模型压缩，如仅进行权重剪枝或仅进行Token剪枝。然而，我们发现这些单一模式的剪枝方法存在显著局限性：首先，不同剪枝模式之间存在复杂的相互依赖关系，单独优化某一模式会错过全局最优解；其次，不同模式对模型性能的影响存在不对称性，某些模式在特定场景下效果更佳。为解决这一问题，我们提出了**协同多模式剪枝（Collaborative Multi-Mode Pruning, CMP）**框架，该框架同时考虑Token剪枝、权重剪枝和架构搜索三种模式，通过联合优化找到全局最优的剪枝配置。CMP的核心创新包括：（1）设计了跨模式的注意力机制，建模不同剪枝模式之间的相互影响；（2）提出了分层搜索策略，在粗粒度和细粒度上分别进行剪枝决策；（3）引入了自适应权重平衡机制，根据训练动态调整各模式的重要性权重。实验表明，CMP在LLaVA-1.5、InstructBLIP等模型上取得了显著效果，在3-5倍压缩比下保持超过85%的原始性能，优于所有单一模式剪枝方法。

## 1. 引言

### 1.1 研究背景

视觉语言模型（VLM）如LLaVA、InstructBLIP和GPT-4V等，通过连接视觉编码器与大语言模型，展现出强大的多模态理解能力。然而，VLM的参数量和计算量巨大，限制了其在资源受限设备上的部署：

| 组件 | 参数量 | 计算量 | 内存占用 |
|-----|-------|-------|---------|
| 视觉编码器 (ViT-L) | 304M | 高 | 大 |
| 连接层 | 10M | 中 | 中 |
| 大语言模型 (7B) | 7B | 极高 | 极大 |

### 1.2 现有方法的局限性

现有的VLM压缩方法主要分为以下几类：

#### （1）权重剪枝（Weight Pruning）

- **方法**：移除神经网络中的冗余参数（权重、神经元、注意力头）
- **代表性工作**：LLM-Pruner [1]、FlashAttention-Merge [2]
- **局限**：对硬件友好性差，稀疏模式难以利用

```
┌─────────────────────────────────────────────────────────────────┐
│                    权重剪枝示意图                                  │
│                                                                     │
│  原始权重矩阵:                                                      │
│  W = [████████████████████████████████]                            │
│                                                                     │
│  剪枝后 (50%稀疏):                                                 │
│  W' = [████░░████░░████░░████░░████░░]                            │
│           ↑        ↑        ↑        ↑                             │
│         稀疏模式，不规则，计算效率低                                │
└─────────────────────────────────────────────────────────────────┘
```

#### （2）Token剪枝（Token Pruning）

- **方法**：移除冗余的视觉Token，减少序列长度
- **代表性工作**：FastV [3]、SaTT [4]、LLaVA-PruMerge [5]
- **局限**：压缩比受限，过于激进的剪枝会丢失关键信息

#### （3）架构搜索（Architecture Search）

- **方法**：搜索最优的模型结构（深度、宽度、头数）
- **代表性工作**：AutoMLP [6]、Vary [7]
- **局限**：搜索空间大，计算成本高

### 1.3 核心观察

我们发现，**单一模式剪枝存在显著的性能瓶颈**：

```
┌─────────────────────────────────────────────────────────────────┐
│  单一模式 vs 协同多模式剪枝                                        │
│                                                                     │
│  单一模式压缩上限:                                                  │
│                                                                     │
│  权重剪枝 50% → 性能损失 5-8%                                      │
│  Token剪枝 50% → 性能损失 3-5%                                     │
│  架构搜索 2x → 性能损失 4-6%                                       │
│                                                                     │
│  协同多模式 (各30%):                                               │
│  权重剪枝 30% + Token剪枝 30% + 架构搜索 30%                       │
│  → 总压缩 ~3x → 性能损失仅 2-3%                                    │
│                                                                     │
│  结论：协同优化可以找到单一模式无法达到的全局最优                    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.4 本文贡献

本文的主要贡献包括：

1. **问题形式化**：首次系统研究VLM中多模式剪枝的协同优化问题。

2. **CMP框架**：提出协同多模式剪枝框架，统一建模Token剪枝、权重剪枝和架构搜索。

3. **跨模式注意力**：设计跨模式的注意力机制，建模不同模式间的相互依赖。

4. **分层搜索策略**：提出粗-细粒度分层搜索，降低搜索复杂度。

5. **实验验证**：在多个VLM和数据集上验证CMP的有效性。

## 3. 方法论

### 3.1 问题定义

给定原始VLM $f(\cdot; W)$，其中 $W$ 是模型参数。我们的目标是找到剪枝后的模型参数 $W'$，使得：

$$\min_{W'} \mathcal{L}_{task}(f(X; W'), Y) \quad \text{s.t.} \quad \text{CR}(W') \geq C$$

其中 $\mathcal{L}_{task}$ 是任务损失，$\text{CR}(\cdot)$ 是压缩比，$C$ 是目标压缩比。

### 3.2 多模式剪枝定义

#### 3.2.1 Token剪枝（TP）

设视觉Token集合为 $\mathbf{V} = \{v_1, ..., v_N\}$，Token剪枝操作定义为：

$$T_{keep} = \text{TokenPrune}(\mathbf{V}; r_t)$$

其中 $r_t \in [0, 1)$ 是Token剪枝比例。剪枝后的Token数量为：

$$|\mathbf{V}'| = N \times (1 - r_t)$$

#### 3.2.2 权重剪枝（WP）

设模型权重矩阵为 $W \in \mathbb{R}^{d_{in} \times d_{out}}$，权重剪枝操作定义为：

$$M = \text{WeightPrune}(W; r_w)$$

其中 $M$ 是二值化掩码，$r_w$ 是权重剪枝比例。剪枝后的权重为：

$$W' = M \odot W$$

#### 3.2.3 架构搜索（AS）

架构搜索操作定义为一组结构参数 $\{\alpha_1, ..., \alpha_K\}$：

$$\text{ArchSearch}(W; \alpha) = \text{AdaptArch}(W, \alpha)$$

### 3.3 协同优化框架

#### 3.3.1 联合优化目标

CMP的优化目标定义为：

$$\min_{\theta_t, \theta_w, \theta_a} \mathcal{L}_{task}(f(X; W(\theta_t, \theta_w, \theta_a)), Y)$$

其中：
- $\theta_t$：Token剪枝参数
- $\theta_w$：权重剪枝参数
- $\theta_a$：架构搜索参数

#### 3.3.2 跨模式注意力机制

不同剪枝模式之间存在复杂的相互依赖：

```
┌─────────────────────────────────────────────────────────────────┐
│  跨模式注意力机制                                                  │
│                                                                     │
│         ┌──────────────┐                                           │
│         │  Token剪枝    │                                           │
│         │  (重要性分)   │                                           │
│         └──────┬───────┘                                           │
│                │                                                    │
│         ┌──────▼───────┐                                           │
│    ┌────┤  Cross-Mode  ├────┐                                       │
│    │    │  Attention   │    │                                       │
│    │    └──────▲───────┘    │                                       │
│    │           │            │                                       │
│ ┌──▼───┐    ┌──▼───┐    ┌──▼───┐                                   │
│ │权重剪│    │跨模式│    │架构  │                                   │
│ │枝    │    │状态  │    │搜索  │                                   │
│ └──────┘    └──────┘    └──────┘                                   │
│                                                                     │
│  注意力计算:                                                         │
│  A_{ij} = softmax(Q_i · K_j / √d) · V_j                            │
│                                                                     │
│  其中Q、K、V来自不同剪枝模式的隐状态                                │
└─────────────────────────────────────────────────────────────────┘
```

### 3.4 分层搜索策略

#### 3.4.1 粗粒度搜索（Layer-level）

首先确定各层的剪枝比例分配：

```
┌─────────────────────────────────────────────────────────────────┐
│  Layer 1: Token剪枝 20% + 权重剪枝 15% → 分配系数 α_1             │
│  Layer 2: Token剪枝 25% + 权重剪枝 20% → 分配系数 α_2            │
│  Layer 3: Token剪枝 30% + 权重剪枝 25% → 分配系数 α_3            │
│  ...                                                             │
│  Layer L: Token剪枝 15% + 权重剪枝 10% → 分配系数 α_L            │
│                                                                     │
│  约束: Σα_l = 1, α_l ≥ 0                                          │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.4.2 细粒度搜索（Token/Weight-level）

在粗粒度分配确定后，进行细粒度的具体剪枝决策：

```python
# 伪代码：细粒度搜索
for layer in model.layers:
    # Token剪枝决策
    token_importance = predict_token_importance(layer.tokens)
    keep_indices = top_k(token_importance, ratio=layer.token_ratio)

    # 权重剪枝决策
    weight_importance = compute_weight_importance(layer.weights)
    mask = top_k_mask(weight_importance, ratio=layer.weight_ratio)

    # 架构搜索决策
    arch_config = search_architecture(layer, task_loss)
```

### 3.5 自适应权重平衡

#### 3.5.1 动态权重更新

各模式的重要性权重在训练过程中动态调整：

$$w_t^{(t)} = \frac{\exp(\mathcal{L}_{TP}^{(t)} / \tau)}{\sum_{m \in \{TP, WP, AS\}} \exp(\mathcal{L}_m^{(t)} / \tau)}$$

其中 $\tau$ 是温度参数，$\mathcal{L}_m^{(t)}$ 是第 $t$ 步各模式的损失贡献。

#### 3.5.2 总损失函数

$$\mathcal{L}_{total} = w_t \cdot \mathcal{L}_{TP} + w_w \cdot \mathcal{L}_{WP} + w_a \cdot \mathcal{L}_{AS}$$

## 5. 结论

本文提出了协同多模式剪枝（CMP）框架，用于视觉语言模型的高效压缩。核心贡献包括：

1. **问题洞察**：首次系统研究VLM中多模式剪枝的协同优化问题，指出单一模式剪枝的局限性。

2. **跨模式注意力**：设计跨模式的注意力机制，建模不同剪枝模式之间的相互依赖关系。

3. **分层搜索策略**：提出粗-细粒度分层搜索，在保证搜索质量的同时降低计算成本。

4. **自适应权重平衡**：引入动态权重调整机制，根据训练进展自适应调整各模式的重要性。

5. **实验验证**：在多个VLM和数据集上验证CMP的有效性，在3-5倍压缩比下保持85%+性能。

### 5.1 局限性

1. **搜索空间**：当前搜索空间仅限于预定义的剪枝比例范围，未来可扩展到更灵活的配置。

2. **任务泛化**：针对特定任务（VQA）优化的剪枝配置可能在其他任务上表现不佳。

3. **训练成本**：多模式协同优化需要更多训练时间和计算资源。

### 5.2 未来工作

1. **跨任务泛化**：研究如何使CMP剪枝的配置泛化到多个下游任务。

2. **硬件感知优化**：考虑不同硬件平台的特性，优化剪枝策略。

3. **动态多模式**：探索推理时动态调整剪枝模式的方法。
