# 通过视觉Token剪枝实现多模态大语言模型的快慢高效训练——详解

| 维度 | 内容 |

| 作者 | Taoyu Chen, Lin Jin, Guangyi Chen 等（北京邮电大学 & 乔治亚理工 & 上海AI Lab） |
| 会议 | EMNLP 2024 Findings |

## 二、核心问题与动机

### 2.1 MLLM训练的高计算成本

训练MLLM的成本主要来自视觉Token的处理：

```
LLaVA-1.5 训练成本分析：
- 图像分辨率：336×336
- ViT编码：576个visual patches（24×24）
- 视觉-文本交叉注意力：576 × seq_len × layers × heads = 海量计算

典型训练配置：
- 预训练：665K图像-文本对
- 微调：150K指令-响应对
- 总计：需要数千GPU小时
```

### 2.2 训练加速的挑战

推理加速方法（如剪枝、量化、蒸馏）无法直接应用于训练，原因：

| 挑战 | 描述 |
|------|------|
| 训练动态复杂 | 训练早期和后期的损失曲面完全不同，早期的剪枝策略不适合后期 |
| 耦合优化问题 | 需要同时优化模型参数和剪枝策略，且两者相互影响 |
| 收敛风险 | 剪枝后的训练可能收敛到次优解 |

### 2.3 关键洞察

> **MLLM训练的不同阶段有不同的信息需求：**
> - **早期阶段**：学习粗粒度的视觉-语言对齐，不需要视觉细节
> - **后期阶段**：学习精细的多模态推理，需要完整视觉信息

这一洞察与人类认知发展规律相似：先学习整体概念，再学习细节。

## 三、方法详解

### 3.1 整体框架

```
训练时间线：
  [=====快速阶段=====][=====慢速阶段=====]
  Epoch 1 → Epoch K       Epoch K+1 → Epoch N
  保留 25-50% Token       保留 75-100% Token
```

**核心设计**：预定义剪枝率调度，不在训练中动态学习剪枝率。

### 3.2 剪枝度量

**视觉显著性（Visual Saliency）**：
$$s_j^{\text{sal}} = \|h_j\|_2$$

Token embedding的L2范数。高范数 = 视觉上更显著的区域（如物体中心）。

**语义相关性（Semantic Relevance）**：
$$s_j^{\text{sem}} = \max_k \text{Attn}_{j,k}$$

Token在语义网络中的重要性，通过最大注意力值度量。

两种度量可以结合使用，或选择其一。实验表明两者效果相近。

### 3.3 预定义剪枝率调度

$$r(t) = r_{\min} + (r_{\max} - r_{\min}) \cdot \frac{t - t_{\min}}{t_{\max} - t_{\min}}$$

其中：
- $t$：当前epoch
- $r_{\min}$：快速阶段的最小保留率（如0.25）
- $r_{\max}$：慢速阶段的最大保留率（如1.0）
- $t_{\min}, t_{\max}$：阶段边界（如5和15）

**关键**：这是一个**确定性调度**，不需要在训练中学习，降低了实现复杂度。

### 3.4 训练稳定性技术

**1. 渐进式热启动（Progressive Warmup）**

前3个epoch：
- Epoch 1-2：保留75% Token（温和剪枝）
- Epoch 3：开始降低到目标剪枝率

作用：让模型先学习基本的视觉-语言对应，再进入快速阶段。

**2. 残差连接（Residual Connection）**

$$\hat{h}_j = h_j + \alpha \cdot \text{Pool}(H)$$

- $h_j$：剪枝后的Token表示
- $\text{Pool}(H)$：全局图像特征的池化（如average pooling）
- $\alpha$：可学习权重，控制残差的贡献

作用：将未剪枝的全局信息传递到后续层，弥补信息损失。

**3. 动态阈值平滑**

$$\bar{r}_t = \beta \cdot r_t + (1 - \beta) \cdot \bar{r}_{t-1}$$

使用EMA平滑剪枝率，避免epoch之间的突变。

### 3.5 与推理阶段剪枝的区别

| 维度 | 推理阶段剪枝 | 训练阶段快慢训练 |
|------|------------|----------------|
| 目标 | 减少推理延迟 | 减少训练计算量 |
| 粒度 | 样本级/层级的Token选择 | 整个训练周期的Token调度 |
| 优化 | 静态剪枝策略 | 动态剪枝率调度 |
| 关键约束 | 保持推理精度 | 保证训练收敛 |

## 四、实验分析

### 4.1 主要结果

| 方法 | GQA | VQAv2 | POPE | MME | 计算减少 |
|------|-----|-------|------|-----|---------|
| LLaVA-1.5 完整 | 62.0 | 80.0 | 85.2 | 1512 | 0% |
| 快慢(25%快速) | 61.8 | 79.8 | 85.0 | 1510 | 38% |
| 快慢(50%快速) | 61.9 | 79.9 | 85.1 | 1511 | 22% |
| 随机剪枝 | 58.4 | 76.2 | 80.3 | 1445 | 38% |

**关键发现**：

1. **重要性引导是关键**：随机剪枝导致3.6%的性能下降，而重要性引导的剪枝几乎没有损失
2. **即使25%也足够**：快速阶段仅保留25%的Token，最终性能几乎不受影响
3. **计算节省显著**：22-38%的训练计算量减少，实际应用中价值巨大

### 4.2 消融实验分析

**渐进式恢复 vs 固定剪枝**：

| 配置 | GQA | 计算减少 |
|------|-----|---------|
| 全程25%固定 | 59.2 | 45% |
| 前5 epoch 25% + 恢复 | 61.8 | 38% |
| 前10 epoch 25% + 恢复 | 61.7 | 42% |

**结论**：
- 全程固定剪枝虽然节省更多计算（45%），但性能损失严重（2.8%）
- 渐进式恢复在计算减少（38%）和性能（仅损失0.2%）之间取得良好平衡
- 前5个epoch快速阶段是最佳选择，更多epoch没有额外收益

**残差连接的作用**：

| 配置 | GQA |
|------|-----|
| 无残差连接 | 60.4 (-1.4) |
| 有残差连接 | 61.8 |

残差连接带来的1.4%提升说明：剪枝确实会造成信息损失，但可以通过残差传递部分弥补。

### 4.3 不同模型规模的适用性

在LLaVA-1.5-7B和Qwen-VL-7B上验证：

- 小模型（7B参数）：快慢训练效果与完整训练相当
- 更大模型：预期效果会更好（因为视觉Token占比更高）

## 五、理论分析

### 5.1 为什么早期可以剪枝？

通过梯度分析发现：

```
早期训练阶段（快速阶段）：
  - 视觉Token梯度主要来自粗粒度特征
  - 细粒度局部纹理的梯度贡献 < 10%
  - 因此剪枝细粒度Token不会显著影响学习

后期训练阶段（慢速阶段）：
  - 需要精细的视觉-语言对齐
  - 局部细节信息变得重要
  - 必须恢复完整Token
```

### 5.2 计算效率分析

**注意力计算的演变**：

| 阶段 | Token数 | 注意力计算复杂度 |
|------|--------|-----------------|
| 完整训练 | 576 + N | O((576+N)^2) |
| 快速阶段(25%) | 144 + N | O((144+N)^2) |
| 慢速阶段(100%) | 576 + N | O((576+N)^2) |

**总计算节省**：
$$\text{savings} = \frac{K \cdot C_{\text{fast}} + (N-K) \cdot C_{\text{slow}}}{N \cdot C_{\text{完整}}}$$

其中 $K$ 为快速阶段的epoch数，$C$ 为各阶段计算量。

典型配置（K=5, N=15, 25%快速阶段）：~35%的总计算节省。

## 六、与其他工作的关系

| 方向 | 代表工作 | 关注点 |
|------|---------|--------|
| 训练加速 | 本文 | MLLM训练的Token剪枝 |
| 推理加速 | TAMP, SparseVLM | 推理时Token剪枝 |
| 训练效率 | LoRA, QLoRA | 参数高效微调，不减少Token |
| Token压缩 | MiniGPT-4, LLaVA-1.5 | 设计更好的视觉编码器 |

本文与推理加速方法（如TAMP）互补：
- **TAMP**：减少推理时的Token，降低延迟
- **本文**：减少训练时的Token，降低训练成本

## 八、实践建议

1. **适用场景**：需要训练自定义MLLM的应用，或大规模MLLM预训练场景
2. **推荐配置**：前5-6个epoch使用25-50%保留率，后8-10个epoch恢复完整Token
3. **必备技术**：残差连接 + 渐进热启动，不要省略
4. **评估方法**：不仅看最终性能，还要看训练损失曲线的收敛速度
5. **进一步优化**：可以结合LoRA等参数高效方法，实现更全面的训练加速
