# 通过视觉上下文压缩实现高效大型多模态模型

**Efficient Large Multimodal Models through Visual Context Compression**

**作者**: Wang, Jiashu and others
**机构**: 多机构合作
**年份**: 2024

---

## 摘要

大型多模态模型(LMM)通过连接视觉编码器与大语言模型实现了强大的多模态理解能力，但视觉token的高计算成本成为瓶颈。本文提出了一种名为**视觉上下文压缩**的方法，通过分析视觉token的内在结构，将相似的token进行聚类和压缩，在保持模型性能的同时显著降低计算成本。实验表明，该方法在保持多模态理解能力的同时，将视觉处理计算量降低了40%，同时在多个基准测试上保持甚至超越了原始模型的性能。

---

## 1. 引言

### 1.1 背景

大型多模态模型(LMM)如LLaVA、MiniGPT-4等，通过将视觉编码器与大语言模型结合，实现了图像理解、视频分析等多模态能力。然而，视觉token的数量庞大（如576个token），带来了显著的内存和计算开销。

### 1.2 核心问题

视觉处理面临的挑战：
1. **token数量多**：单张图像可能产生数百个token
2. **信息冗余**：相邻像素的信息高度相似
3. **计算密集**：自注意力随token数量呈二次增长

### 1.3 本文贡献

1. **视觉上下文分析**：深入分析视觉token间的信息冗余
2. **聚类压缩方法**：将相似token聚类并压缩
3. **保持语义完整**：确保压缩后不丢失关键视觉信息

## 3. 方法

### 3.1 视觉上下文分析

#### 3.1.1 token相似性度量

$$sim(t_i, t_j) = \frac{t_i \cdot t_j}{\|t_i\| \|t_j\|}$$

#### 3.1.2 信息冗余检测

高相似度的token对暗示了信息冗余：
- 空间相邻区域的token相似度高
- 相似纹理/颜色区域的token相似度高

### 3.2 聚类压缩

#### 3.2.1 K-means聚类

将视觉token按特征相似度聚类：
$$\min_{C} \sum_{i=1}^{K} \sum_{t \in C_i} \|t - \mu_i\|^2$$

其中 $\mu_i$ 是聚类中心。

#### 3.2.2 代表token选择

每个聚类选择一个代表token：
$$t_{rep} = \arg\max_{t \in C_i} \sum_{t' \in C_i} sim(t, t')$$

### 3.3 压缩流程

```
输入：视觉token序列 T = [t₁, t₂, ..., tₙ]
输出：压缩后token序列 T' = [t'₁, t'₂, ..., t'ₖ]

1. # 计算相似度矩阵
2. S = similarity_matrix(T)
3.
4. # K-means聚类
5. clusters = kmeans(T, K)
6.
7. # 生成代表token
8. T' = [representative(c) for c in clusters]
9.
10. return T'
```

## 4. 实验

### 4.1 实验设置

**模型**：LLaVA-1.5-7B
**数据集**：VQA-v2, GQA, ScienceQA
**压缩比例**：2x, 4x, 8x

### 4.2 主要结果

| 方法 | VQAv2 | GQA | 压缩比 |
|------|-------|-----|--------|
| 原始 | 80.0% | 63.8% | 1x |
| LLaVA | 77.5% | 61.2% | 2x |
| **本文方法** | **79.2%** | **63.1%** | **2x** |
