# 面向大型视觉语言模型的拓扑感知层剪枝

## Topology-Aware Layer Pruning for Large Vision-Language Models

**作者**: Shuteng Liu, Kai Wang, Lin Yue, Chong Chen, Xiao Wang, Guoren Wang
**机构**: 北京理工大学、中国科学院计算技术研究所
**发表**: ACL 2024 (Findings)
**arXiv**: 2406.11523

## 摘要

大型视觉语言模型（VLM）如LLaVA和InstructBLIP已成为多模态理解的基础架构。然而，这些模型的Transformer层数众多（如LLaVA-1.5包含32层），导致推理时的计算和内存开销巨大。传统的层剪枝方法通常独立评估每层的重要性，然后移除低重要性层，但这种方法忽视了层与层之间的拓扑依赖关系。实际上，VLM中的层并非独立存在，它们通过注意力机制形成复杂的拓扑结构，层间的信息流动具有很强的依赖性。盲目移除低重要性层可能破坏这种拓扑结构，导致信息流中断和性能显著下降。为解决这一问题，我们提出了**TALP（Topology-Aware Layer Pruning，拓扑感知层剪枝）**方法。TALP的核心思想是在评估层重要性时，不仅考虑单层的独立贡献，还要考虑该层在整体拓扑结构中的位置和作用。具体而言，我们首先构建VLM的层间拓扑图，通过图分析识别层间的依赖关系和功能模块；然后根据拓扑结构和功能角色对层进行分组，确保剪枝操作不会破坏重要的信息传递路径；最后在每个分组内选择代表性层进行保留，移除功能冗余的层。实验表明，TALP在LLaVA-1.5（7B和13B）等模型上实现了30%-50%的层剪枝，同时保持90%以上的原始性能，显著优于随机剪枝和独立重要性剪枝等基线方法。

## 1. 引言

### 1.1 研究背景

大型视觉语言模型（VLM）通过连接视觉编码器与大语言模型，实现了强大的视觉理解能力。以LLaVA-1.5为例，其架构包含：

```
┌─────────────────────────────────────────────────────────────────┐
│                    LLaVA-1.5 架构图                              │
│                                                                     │
│  输入图像                                                            │
│      ↓                                                              │
│  CLIP ViT-L/14 (24层)                                              │
│      ↓                                                              │
│  连接层 (Projection)                                                │
│      ↓                                                              │
│  Vicuna LLM (32层)                                                 │
│      ↓                                                              │
│  输出响应                                                            │
└─────────────────────────────────────────────────────────────────┘
```

虽然视觉编码器层数相对较少（通常24层），但大语言模型的层数往往很多（7B模型通常32层，13B模型可达40层）。

### 1.2 层剪枝的动机

层剪枝（Layer Pruning）是一种有效的模型压缩技术，其优势包括：

| 压缩方式 | 参数量减少 | 计算量减少 | 内存减少 |
|---------|-----------|-----------|---------|
| 权重剪枝 | 中 | 低 | 中 |
| Token剪枝 | 无 | 中 | 高 |
| **层剪枝** | **高** | **高** | **高** |
| 知识蒸馏 | 中 | 中 | 中 |

### 1.3 现有方法的局限性

#### （1）独立重要性评估

传统方法独立评估每层的重要性：

```
┌─────────────────────────────────────────────────────────────────┐
│                    独立重要性评估                                 │
│                                                                     │
│  Layer 1: 重要性 0.45 (保留)                                        │
│  Layer 2: 重要性 0.52 (保留)                                        │
│  Layer 3: 重要性 0.38 (剪枝?) ← 忽略拓扑依赖                        │
│  Layer 4: 重要性 0.48 (保留)                                        │
│  Layer 5: 重要性 0.41 (剪枝?) ← 忽略信息流                          │
│  ...                                                               │
│                                                                     │
│  问题：如果Layer 3和Layer 5是关键信息传递节点，移除它们会破坏信息流 │
└─────────────────────────────────────────────────────────────────┘
```

**问题**：这种方法忽视了层间的信息传递依赖。

#### （2）随机剪枝

随机移除若干层，结果通常很差。

#### （3）均匀间隔剪枝

每间隔 $k$ 层移除一层，但这种方法没有考虑层的实际功能。

### 1.4 核心观察：拓扑依赖性

我们观察到VLM的层间存在显著的拓扑依赖性：

```
┌─────────────────────────────────────────────────────────────────┐
│                    层间拓扑依赖示意图                              │
│                                                                     │
│  信息流：                                                            │
│  Layer 1 → Layer 2 → Layer 3 → ... → Layer 32                      │
│    ↓           ↓           ↓                ↓                       │
│  低层特征    中层特征    高层特征        语义表示                    │
│  (纹理、边缘)  (物体部件)  (完整物体)     (概念、推理)                 │
│                                                                     │
│  关键发现：                                                          │
│  - 某些层是"信息传递节点"，移除它们会中断信息流                     │
│  - 相邻层往往功能相似，可以分组处理                                 │
│  - 信息流具有层级特性，高层依赖低层                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.5 本文贡献

1. **拓扑分析框架**：首次系统分析VLM的层间拓扑结构，揭示层间的依赖关系。

2. **TALP方法**：提出拓扑感知层剪枝方法，考虑层在拓扑结构中的位置。

3. **功能分组机制**：设计基于功能角色的层分组策略，确保剪枝不破坏信息流。

4. **分组剪枝策略**：在每个分组内进行均匀剪枝，保持组内功能覆盖。

5. **实验验证**：在多个VLM上验证TALP的有效性。

## 3. 方法论

### 3.1 问题定义

给定一个具有 $L$ 层Transformer的VLM：

$$f = \{h_1, h_2, ..., h_L\}$$

其中 $h_l(\cdot)$ 是第 $l$ 层的变换函数。

**目标**：找到一个层子集 $\mathcal{P} \subset \{1, 2, ..., L\}$，使得 $|\mathcal{P}| = M < L$，同时最大化下游任务性能：

$$\min_{\mathcal{P}} \mathcal{L}_{task}(f_{\mathcal{P}}(X), Y) \quad \text{s.t.} \quad |\mathcal{P}| = M$$

其中 $f_{\mathcal{P}}$ 表示保留层的子集。

### 3.2 层间拓扑图构建

#### 3.2.1 拓扑图定义

我们构建层间拓扑图 $G = (V, E)$：

- **节点** $V = \{v_1, v_2, ..., v_L\}$：每个节点代表一层
- **边** $E$：边的权重表示两层之间的信息传递强度

#### 3.2.2 信息传递强度计算

两层之间的信息传递强度通过以下方式计算：

$$w_{ij} = \frac{1}{T} \sum_{t=1}^{T} \text{AttentionScore}_{layer=i \to j}^{(t)}$$

其中 $T$ 是样本数量，$\text{AttentionScore}_{layer=i \to j}^{(t)}$ 是第 $t$ 个样本中第 $i$ 层到第 $j$ 层的注意力分数。

```
┌─────────────────────────────────────────────────────────────────┐
│                    层间拓扑图示例                                │
│                                                                     │
│           Layer 1                                                  │
│          /      \                                                  │
│        /          \                                                │
│    Layer 2    Layer 3 ← 相邻层连接强                               │
│       |  \      /  \                                               │
│       |    \  /     \                                              │
│    Layer 4 ---- Layer 5 ← 信息汇聚节点                             │
│          \        /                                                │
│           \      /                                                 │
│          Layer 6 ← 功能转变层                                       │
│                                                                     │
│  边的粗细表示信息传递强度                                           │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 拓扑重要性评估

#### 3.3.1 入度重要性

节点接收到的信息量：

$$I_l^{in} = \sum_{i=1}^{L} w_{il}$$

#### 3.3.2 出度重要性

节点发送出的信息量：

$$I_l^{out} = \sum_{j=1}^{L} w_{lj}$$

#### 3.3.3 拓扑中心性

综合考虑入度和出度：

$$I_l^{topo} = \alpha \cdot I_l^{in} + \beta \cdot I_l^{out} + \gamma \cdot C_l^{between}$$

其中 $C_l^{between}$ 是介数中心性（Betweenness Centrality），定义为：

$$C_l^{between} = \sum_{s \neq l \neq t} \frac{\sigma_{st}(l)}{\sigma_{st}}$$

$\sigma_{st}$ 是从节点 $s$ 到节点 $t$ 的最短路径数，$\sigma_{st}(l)$ 是经过节点 $l$ 的最短路径数。

### 3.4 功能角色分析

#### 3.4.1 层级功能假设

我们假设VLM的层具有层级功能特性：

| 层级范围 | 功能描述 | 特征类型 |
|---------|---------|---------|
| 底层 (Layer 1-8) | 视觉特征提取 | 局部特征、纹理、边缘 |
| 中层 (Layer 9-20) | 概念抽象 | 物体部件、空间关系 |
| 高层 (Layer 21-32) | 语义理解 | 完整物体、动作、场景 |

#### 3.4.2 层功能相似度

我们使用余弦相似度衡量层间的功能相似性：

$$\text{sim}(h_i, h_j) = \frac{\text{Act}_i \cdot \text{Act}_j}{\|\text{Act}_i\| \cdot \|\text{Act}_j\|}$$

其中 $\text{Act}_i$ 是第 $i$ 层在验证集上的激活向量。

#### 3.4.3 功能分组

基于功能相似度，使用层次聚类将层分为 $K$ 个功能组：

$$G_k = \{l \mid \text{Cluster}(h_l) = k\}, \quad k = 1, 2, ..., K$$

```
┌─────────────────────────────────────────────────────────────────┐
│                    功能分组示意图                                  │
│                                                                     │
│  底层组 (K=4):                                                      │
│  [Layer 1] [Layer 2] [Layer 3] [Layer 4] ← 功能相似，可分组剪枝    │
│                                                                     │
│  中层组 (K=8):                                                      │
│  [Layer 5] [Layer 6] [Layer 7] [Layer 8] [Layer 9] ...            │
│                                                                     │
│  高层组 (K=4):                                                      │
│  ... [Layer 29] [Layer 30] [Layer 31] [Layer 32]                  │
│                                                                     │
│  策略：每组均匀剪枝，确保各功能层级都有代表                         │
└─────────────────────────────────────────────────────────────────┘
```

### 3.5 拓扑感知剪枝算法

#### 3.5.1 两阶段剪枝

**阶段1：拓扑分析**

1. 计算层间信息传递强度矩阵 $W$
2. 计算每层的拓扑重要性 $I_l^{topo}$
3. 基于功能相似度进行层次聚类，得到 $K$ 个功能组

**阶段2：分组剪枝**

对于每个功能组 $G_k$，按比例 $\rho$ 进行均匀剪枝：

```
┌─────────────────────────────────────────────────────────────────┐
│                    分组剪枝算法                                   │
│                                                                     │
│  输入: 功能组 G_k = {l_1, l_2, ..., l_m}，剪枝比例 ρ              │
│  输出: 保留的层集合                                                 │
│                                                                     │
│  1. 计算组内每层的拓扑重要性:                                       │
│     for l in G_k:                                                  │
│         I_l^{topo} = α·I_l^{in} + β·I_l^{out} + γ·C_l^{between}   │
│                                                                     │
│  2. 按重要性排序:                                                  │
│     sorted_layers = sort(G_k, by=I_l^{topo})                      │
│                                                                     │
│  3. 均匀采样剪枝位置:                                              │
│     step = floor(|G_k| / (|G_k|·ρ))                               │
│     prune_indices = range(0, |G_k|, step)                         │
│                                                                     │
│  4. 返回保留的层                                                    │
│     return G_k \ {prune_indices}                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.5.2 剪枝比例分配

不同功能组的剪枝比例可以不同：

| 功能组 | 层范围 | 建议剪枝比例 | 原因 |
|-------|-------|-------------|------|
| 低层组 | 1-8 | 30% | 冗余度高，功能相似 |
| 中层组 | 9-20 | 20% | 功能变化，冗余度中等 |
| 高层组 | 21-32 | 15% | 语义关键，需谨慎 |

### 3.6 权重调整

剪枝后需要对保留的层进行权重调整，以恢复性能：

$$\mathcal{L}_{ft} = \mathcal{L}_{task} + \lambda \cdot \mathcal{L}_{topo}$$

其中 $\mathcal{L}_{topo}$ 是拓扑一致性损失：

$$\mathcal{L}_{topo} = \sum_{(i,j) \in E} w_{ij} \cdot \|\text{Act}_i - \text{Act}_j\|^2$$

## 4. 实验

### 4.1 实验设置

#### 4.1.2 基准数据集

| 数据集 | 任务类型 | 样本数 |
|-------|---------|-------|

#### 4.1.3 对比方法

| 方法 | 描述 |
|------|------|
| **No-Prune** | 不进行层剪枝（基线） |

| **Uniform** | 均匀间隔移除层 |
| **L2-Norm** | 基于层输出范数的重要性剪枝 |

### 4.2 主要结果

#### 4.2.1 VQAv2数据集

| 方法 | 保留层数 | VQAv2 | 参数量减少 |
|------|---------|-------|-----------|

| L2-Norm | 22/32 | 75.2% | 31% |
| **TALP (Ours)** | **22/32** | **76.8%** | **31%** |

**关键发现**：TALP在相同剪枝比例下，性能显著优于所有基线方法。

#### 4.2.2 不同剪枝比例

| 剪枝比例 | Random | Uniform | L2-Norm | TALP |
|---------|--------|--------|--------|------|

|------|--------|---------|

#### 4.3.1 拓扑信息贡献

| 变体 | VQAv2 | 分析 |
|------|-------|------|
| 无拓扑信息 (仅功能分组) | 75.4% | 忽视拓扑依赖 |

## 5. 结论

本文提出了TALP（Topology-Aware Layer Pruning），一种考虑层间拓扑结构的VLM层剪枝方法。核心贡献包括：

1. **拓扑分析**：首次系统分析VLM的层间拓扑结构，揭示层间的依赖关系。

2. **功能分组**：设计基于功能相似度的层分组策略，确保剪枝不破坏功能覆盖。

3. **TALP算法**：提出综合考虑拓扑重要性和功能分组的剪枝算法。

4. **实验验证**：在多个VLM和数据集上验证TALP的有效性，在30%层剪枝下保持90%+性能。

### 5.1 局限性

1. **分组策略**：当前使用层次聚类，可能不是最优的功能分组方法。

2. **静态剪枝**：剪枝比例固定，无法根据输入动态调整。

3. **与Token剪枝的结合**：未考虑层剪枝与Token剪枝的联合优化。

### 5.2 未来工作

1. **自适应分组**：根据输入动态调整功能分组。

2. **联合剪枝**：将层剪枝与Token剪枝、权重剪枝结合。

3. **知识蒸馏**：结合知识蒸馏进一步恢复剪枝后的性能。
