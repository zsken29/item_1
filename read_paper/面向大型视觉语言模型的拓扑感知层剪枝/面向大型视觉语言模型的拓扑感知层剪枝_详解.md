# 面向大型视觉语言模型的拓扑感知层剪枝 — 详解

**TALP: Topology-Aware Layer Pruning for Large Vision-Language Models**

**作者**: Shuteng Liu, Kai Wang, Lin Yue, Chong Chen, Xiao Wang, Guoren Wang
**机构**: 北京理工大学、中国科学院计算技术研究所
**发表**: ACL 2024 (Findings)
**arXiv**: 2406.11523

---

## ★ Insight（关键洞察）

1. **拓扑依赖建模**：首次提出VLM层间的拓扑依赖关系，利用连接图分析每层在信息流中的位置，剪枝时保持拓扑完整性

2. **功能角色分组**：发现VLM不同层承担不同功能角色（如底层特征提取、中层语义融合、高层任务推理），同功能层可合并剪枝

3. **信息流保持原则**：剪枝不应破坏模型的信息流完整性，TALP通过拓扑分析确保关键信息路径不被切断

4. **组内代表性剪枝**：同组相似层中保留代表性层，剪枝冗余层，比单独评估每层更有效

5. **30%层剪枝保持90%+性能**：在显著减少参数量（30%）的同时保持良好性能，证明拓扑感知剪枝的有效性

## 一、论文核心故事线（先读这里）

**一句话总结**：提出TALP方法，通过分析VLM层间的拓扑依赖和功能角色，将相似层分组后剪枝，在保持信息流完整性的同时实现30%层剪枝，保留90%+性能。

**核心创新点**：
- 层间拓扑图构建
- 功能角色分析
- 组内代表性剪枝
- 拓扑感知优化

**技术路径**：拓扑分析 → 功能聚类 → 分组剪枝 → 性能保持

## 二、研究背景与动机

### 2.1 VLM的层结构

VLM通常包含多层Transformer：
- 视觉编码器层（ViT）
- 跨模态融合层
- 语言解码器层（LLM）

以LLaVA-1.5为例：
```
输入图像
    ↓
CLIP ViT-L/14 (24层)
    ↓
连接层 (Projection)
    ↓
Vicuna LLM (32层)
    ↓
输出响应
```

### 2.2 层剪枝的动机

层剪枝（Layer Pruning）是一种有效的模型压缩技术：

| 压缩方式 | 参数量减少 | 计算量减少 | 内存减少 |
|---------|-----------|-----------|---------|
| 权重剪枝 | 中 | 低 | 中 |
| Token剪枝 | 无 | 中 | 高 |
| **层剪枝** | **高** | **高** | **高** |

### 2.3 现有方法的局限性

**独立重要性评估**：
- 传统方法独立评估每层的重要性
- 忽视了层间的信息传递依赖
- 可能破坏拓扑结构

**随机剪枝**：
- 随机移除若干层，结果通常很差

**均匀间隔剪枝**：
- 每间隔k层移除一层
- 没有考虑层的实际功能

### 2.4 核心观察：拓扑依赖性

VLM的层间存在显著的拓扑依赖性：

```
信息流：
Layer 1 → Layer 2 → Layer 3 → ... → Layer 32
  ↓          ↓          ↓                ↓
低层特征   中层特征   高层特征        语义表示
(纹理、边缘)  (物体部件)  (完整物体)     (概念、推理)

关键发现：
- 某些层是"信息传递节点"，移除它们会中断信息流
- 相邻层往往功能相似，可以分组处理
- 信息流具有层级特性，高层依赖低层
```

## 三、方法详解（含公式）

### 3.1 问题定义

给定一个具有 $L$ 层Transformer的VLM：

$$f = \{h_1, h_2, ..., h_L\}$$

**目标**：找到一个层子集 $\mathcal{P} \subset \{1, 2, ..., L\}$，使得 $|\mathcal{P}| = M < L$：

$$\min_{\mathcal{P}} \mathcal{L}_{task}(f_{\mathcal{P}}(X), Y) \quad \text{s.t.} \quad |\mathcal{P}| = M$$

### 3.2 层间拓扑图构建

#### 拓扑图定义

构建层间拓扑图 $G = (V, E)$：
- **节点** $V = \{v_1, v_2, ..., v_L\}$：每个节点代表一层
- **边** $E$：边的权重表示两层之间的信息传递强度

#### 信息传递强度计算

两层之间的信息传递强度：

$$w_{ij} = \frac{1}{T} \sum_{t=1}^{T} \text{AttentionScore}_{layer=i \to j}^{(t)}$$

### 3.3 拓扑重要性评估

#### 入度重要性

节点接收到的信息量：

$$I_l^{in} = \sum_{i=1}^{L} w_{il}$$

#### 出度重要性

节点发送出的信息量：

$$I_l^{out} = \sum_{j=1}^{L} w_{lj}$$

#### 拓扑中心性

综合考虑入度和出度：

$$I_l^{topo} = \alpha \cdot I_l^{in} + \beta \cdot I_l^{out} + \gamma \cdot C_l^{between}$$

其中 $C_l^{between}$ 是介数中心性：

$$C_l^{between} = \sum_{s \neq l \neq t} \frac{\sigma_{st}(l)}{\sigma_{st}}$$

### 3.4 功能角色分析

#### 层级功能假设

| 层级范围 | 功能描述 | 特征类型 |
|---------|---------|---------|
| 底层 (Layer 1-8) | 视觉特征提取 | 局部特征、纹理、边缘 |
| 中层 (Layer 9-20) | 概念抽象 | 物体部件、空间关系 |
| 高层 (Layer 21-32) | 语义理解 | 完整物体、动作、场景 |

#### 层功能相似度

使用余弦相似度衡量层间的功能相似性：

$$\text{sim}(h_i, h_j) = \frac{\text{Act}_i \cdot \text{Act}_j}{\|\text{Act}_i\| \cdot \|\text{Act}_j\|}$$

### 3.5 功能分组

基于功能相似度，使用层次聚类将层分为 $K$ 个功能组：

$$G_k = \{l \mid \text{Cluster}(h_l) = k\}, \quad k = 1, 2, ..., K$$

### 3.6 分组剪枝策略

对于每个功能组 $G_k$，按比例 $\rho$ 进行均匀剪枝：

```
输入: 功能组 G_k = {l_1, l_2, ..., l_m}，剪枝比例 ρ
输出: 保留的层集合

1. 计算组内每层的拓扑重要性:
   I_l^{topo} = α·I_l^{in} + β·I_l^{out} + γ·C_l^{between}

2. 按重要性排序:
   sorted_layers = sort(G_k, by=I_l^{topo})

3. 均匀采样剪枝位置:
   step = floor(|G_k| / (|G_k|·ρ))
   prune_indices = range(0, |G_k|, step)

4. 返回保留的层
   return G_k \ {prune_indices}
```

### 3.7 剪枝比例分配

| 功能组 | 层范围 | 建议剪枝比例 | 原因 |
|-------|-------|-------------|------|
| 低层组 | 1-8 | 30% | 冗余度高，功能相似 |
| 中层组 | 9-20 | 20% | 功能变化，冗余度中等 |
| 高层组 | 21-32 | 15% | 语义关键，需谨慎 |

## 六、方法优缺点深度评析

### 6.1 优点

1. **拓扑感知**：考虑层间连接结构，保持信息流完整性
2. **功能分组**：区分层功能，避免破坏关键功能
3. **性能保持优秀**：30%层剪枝保持96%+性能
4. **可解释性强**：拓扑图提供直观的层重要性理解

### 6.2 缺点

1. **分组策略**：当前使用层次聚类，可能不是最优的
2. **静态剪枝**：剪枝比例固定，无法根据输入动态调整
3. **与Token剪枝的结合**：未考虑层剪枝与Token剪枝的联合优化

### 6.3 潜在改进方向

1. **自适应分组**：根据输入动态调整功能分组
2. **联合剪枝**：将层剪枝与Token剪枝、权重剪枝结合
3. **知识蒸馏**：结合知识蒸馏进一步恢复剪枝后的性能
