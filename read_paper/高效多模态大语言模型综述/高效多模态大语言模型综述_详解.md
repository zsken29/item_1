### 2.2 视觉编码器高效化

**轻量化视觉编码器**：
- MobileViT：移动端优化的Vision Transformer
- EfficientViT：重参数化+多尺度特征
- TinyViT：专门压缩的小型ViT

**视觉编码器剪枝**：
- 结构化剪枝：移除注意力头、FFN层
- 知识蒸馏：从小编码器迁移知识
- 动态Token：按重要性选择视觉Token

**视觉编码器适配**：
- 线性投影层优化：减小模态差距
- 对齐预训练：CLIP式对比学习
- 提示学习：Visual Prompt Tuning

### 2.3 模态融合高效化

**早期融合**：
- 在输入层融合视觉与文本
- 减少融合计算开销
- 代表：Flamingo、LLaVA

**晚期融合**：
- 各模态独立处理后融合
- 灵活性高但计算量大
- 代表：GPT-4V

**混合融合策略**：
- 根据任务自适应选择融合时机
- 粗粒度早期融合 + 细粒度晚期精修

### 2.4 LLM骨干高效化

**小模型替代**：
- LLaMA 7B → 3B → 1B
- Phi-3小模型高性能路线
- MiniCPM等国产小模型

**专家混合（MoE）**：
- 稀疏激活减少计算
- 专家数量可扩展
- 代表：Mixtral-8x7B

**长上下文优化**：
- 稀疏注意力
- 近似注意力计算
- KV Cache优化

### 2.5 训练高效化

**参数高效微调（PEFT）**：
| 方法 | 原理 | 参数量 | 效果 |
|------|------|--------|------|
| LoRA | 低秩分解 | ~0.1% | 优秀 |
| Adapter | bottleneck adapter | ~1-5% | 良好 |
| Prefix-tuning | 前缀向量 | ~0.1% | 良好 |
| Prompt-tuning | 软提示 | ~0.01% | 中等 |

**数据高效利用**：
- 数据筛选：选择高质量样本
- 数据增强：多模态增强策略
- 课程学习：渐进式训练

### 2.6 推理高效化

**Token剪枝**：
- 重要性评估：基于注意力、梯度、激活值
- 动态剪枝：根据输入自适应
- 结构化剪枝：按组剪枝

**推理缓存**：
- KV Cache优化：减少重复计算
- 提示缓存：复用常见模式
- 语义缓存：基于语义相似度

**早退出机制**：
- 浅层退出：简单任务浅层决策
- 动态深度：根据置信度选择
- 级联推理：多模型级联

### 3.2 技术路线对比

**路线一：模型层面压缩**
- 优势：推理加速直接彻底
- 劣势：可能有精度损失
- 适用：资源受限场景

**路线二：推理层面优化**
- 优势：不改变模型结构
- 劣势：加速效果有限
- 适用：快速部署场景

**路线三：训练层面优化**
- 优势：可以保持精度
- 劣势：需要重新训练
- 适用：新模型设计

## 四、关键技术原理

### 4.1 视觉Token压缩原理

**基于重要性的评估**：
$$ score_i = \alpha \cdot attn_i + \beta \cdot act_i + \gamma \cdot grad_i $$

综合注意力、激活值、梯度等多信号评估Token重要性。

**基于重构的评估**：
训练轻量网络预测被丢弃Token的信息，保持语义完整性。

### 4.2 模态对齐原理

**对比学习对齐**：
$$ \mathcal{L}_{align} = -\log \frac{\exp(sim(v_i, t_i)/\tau)}{\sum_j \exp(sim(v_i, t_j)/\tau)} $$

拉近匹配视觉-文本表示，拉远不匹配对。

**Adapter对齐**：
在冻结的编码器间插入小型Adapter，实现轻量适配。

## 六、实践建议

### 6.1 方法选择决策树

```
需要高效化MLLM？
├── 资源极度受限？
│   ├── 是 → 模型层面压缩（小模型/MoE）
│   └── 否 → 继续判断
├── 是否需要微调？
│   ├── 是 → PEFT（LoRA/QLoRA）
│   └── 否 → 推理层面优化
└── 任务是否特定？
    ├── 是 → 任务适配 + Token剪枝
    └── 否 → 通用高效化策略
```

### 6.2 部署场景推荐

| 场景 | 推荐方法 |
|------|----------|
| 云端服务器 | INT8量化 + KV Cache优化 |
| 边缘设备 | INT4量化 + 模型蒸馏 |
| 移动应用 | 小模型 + 动态Token剪枝 |
| 嵌入式 | 模型压缩 + 极致量化 |

## 七、总结

本文是一篇全面的MLLM高效化综述，覆盖架构、训练、推理三个层面的优化方法。论文提供了系统化的分类框架，便于研究者理解各方法之间的关系和适用场景。对于VLM压缩研究，本文提供了重要的技术全景视图，有助于选择合适的压缩策略。

**核心 takeaways**：
1. 高效化需要从多个层面综合考虑
2. 推理层面优化适合快速部署，模型层面压缩适合资源受限场景
3. Token剪枝是多模态特有的高效化手段
4. 方法组合往往优于单一方法
