# CoreMatching：通过Token与神经元联合剪枝实现视觉语言模型全面加速 — 详解

**CoreMatching: Collaborative Adaptive Sparse Inference Framework via Joint Token and Neuron Pruning**

---

## ★ Insight（关键洞察）

1. **联合剪枝范式**：首次提出同时剪枝视觉Token和语言神经元，填补了Token剪枝和权重剪枝之间的空白，实现全面的模型压缩

2. **耦合关系建模**：发现Token和神经元存在重要性耦合关系，通过跨模态相关性量化这种依赖，联合优化避免各自为政的次优解

3. **迭代协同优化**：采用交替优化的迭代框架，每次固定一个维度优化另一个，逐步收敛到联合最优解

4. **全面加速覆盖**：同时加速视觉编码和语言推理两个阶段，而非仅优化单一瓶颈

5. **性能保持优秀**：联合剪枝比单独剪枝效果更好，因为可以协调两个维度的重要性评估

## 二、研究背景与动机

### 2.1 VLM的双阶段计算

VLM的计算分布在两个阶段：
- **视觉编码**：视觉token处理，O(n²)复杂度
- **语言推理**：语言模型推理，显存密集

### 2.2 单一维度剪枝的问题

| 方法 | 优化维度 | 问题 |
|------|---------|------|
| Token剪枝 | 视觉 | 语言阶段未优化 |
| 权重剪枝 | 语言 | 视觉阶段未优化 |
| 各自为政 | 双维度 | 忽视耦合关系 |

### 2.3 核心洞察

**Token-神经元耦合假设**：
- 重要Token的注意力会导向重要神经元
- 重要神经元处理来自重要Token的信息
- 两者应联合评估，而非独立剪枝

### 2.4 核心动机

设计全面优化的剪枝方法：
- **双维度**：同时剪枝Token和神经元
- **协同**：考虑两者的耦合关系
- **全面**：覆盖视觉和语言两阶段
- **高效**：保持性能的同时加速

## 三、核心问题分析

### 3.1 问题定义

给定VLM $f$ 和资源约束 $C$，同时确定Token剪枝比例 $r_t$ 和神经元剪枝比例 $r_n$：

$$\min_{r_t, r_n} \mathcal{L}(f(T^{r_t}, \theta^{r_n}))$$

约束：$\text{Latency}(f) \leq C$

### 3.2 技术挑战

**挑战1：耦合关系量化**
- Token和神经元如何相互影响？
- 如何衡量它们的相关性？
- **解决方案**：梯度相关性

**挑战2：联合优化求解**
- 两个维度的组合空间巨大
- 交替优化可能陷入局部最优
- **解决方案**：迭代优化+动量

**挑战3：性能平衡**
- Token剪枝过度 → 视觉信息丢失
- 神经元剪枝过度 → 语义理解下降
- **解决方案**：联合损失约束

## 四、方法详解（含公式）

### 4.1 Token重要性评估

#### 4.1.1 注意力感知重要性

基于多头注意力的重要性：
$$I_i^t = \sum_{h=1}^{H} \sum_{l=1}^{L} \alpha_{i \leftarrow j}^{l,h} \cdot v_j^{l,h}$$

token $i$ 的重要性与其收到的总注意力成正比。

#### 4.1.2 激活感知重要性

$$I_i^t = |t_i| \cdot \|a_i\|$$

结合token大小和激活强度。

### 4.2 神经元重要性评估

#### 4.2.1 Fisher信息

$$I_j^n = \mathbb{E}_x[(\frac{\partial \log f(x)}{\partial \theta_j})^2]$$

#### 4.2.2 激活感知

$$I_j^n = |W_j| \cdot \|x_j\|$$

### 4.3 跨模态相关性

#### 4.3.1 梯度相关性定义

$$corr(t_i, \theta_j) = \mathbb{E}[(\frac{\partial \mathcal{L}}{\partial t_i}) \cdot (\frac{\partial \mathcal{L}}{\partial \theta_j})]$$

衡量token $i$ 和神经元 $j$ 对损失的联合贡献。

#### 4.3.2 耦合效应

高相关性的token-neuron对应同时保留：
- 保留重要token配合重要神经元
- 剪枝不重要token配合不重要神经元
- 避免"错配"导致的性能损失

### 4.4 迭代协同优化

#### 4.4.1 交替优化框架

```
Algorithm: CoreMatching Optimization
Input: VLM f, token ratio rt, neuron ratio rn
Output: Token mask mt, neuron mask mn

1. # 初始化
2. mt = ones(n)  # 全部保留
3. mn = ones(m)  # 全部保留

4. # 迭代优化
5. for iteration in range(N):
6.     # Step 1: 固定神经元，优化Token
7.     for i in range(n):
8.         score_t[i] = I_i^t * (1 + γ * Σj corr(t_i, θ_j) * mn[j])
9.     mt = TopK(score_t, 1-rt)

10.    # Step 2: 固定Token，优化神经元
11.    for j in range(m):
12.        score_n[j] = I_j^n * (1 + γ * Σi corr(t_i, θ_j) * mt[i])
13.    mn = TopK(score_n, 1-rn)

14.    # Step 3: 检查收敛
15.    if converged(mt, mn):
16.        break

17. return mt, mn
```

#### 4.4.2 耦合损失项

$$\mathcal{L}_{total} = \mathcal{L}_{task} + \lambda_t \|m^t\|_0 + \lambda_n \|m^n\|_0 + \gamma \mathcal{L}_{coupling}$$

其中耦合损失：
$$\mathcal{L}_{coupling} = \sum_{i,j} corr(t_i, \theta_j) \cdot (1 - m_i^t) \cdot (1 - m_j^n)$$

鼓励保留高相关性的组合。

### 4.5 剪枝执行

#### 4.5.1 Token剪枝

$$T_{pruned} = T[mt = 1]$$

#### 4.5.2 神经元剪枝

$$W_{pruned} = W[mn = 1, mn = 1]$$

## 七、与相关工作的关系

### 7.1 与Token剪枝方法的关系

SparseVLM、EfficientLLaVA等方法仅关注Token剪枝：
- 仅优化视觉阶段
- 未考虑语言模型优化
- 可能导致Token剪枝过度

CoreMatching扩展了Token剪枝：
- 同时优化语言神经元
- 协调两个维度
- 更全面的加速

### 7.2 与权重剪枝方法的关系

LLM-Pruner、Wanda等方法仅关注权重剪枝：
- 仅优化语言阶段
- 未考虑视觉优化
- 可能忽视关键视觉信息

CoreMatching扩展了权重剪枝：
- 同时优化视觉Token
- 全面的模型压缩
- 性能更优

### 7.3 与联合压缩的关系

CoreMatching采用交替优化而非端到端优化：
- 简化优化难度
- 便于收敛
- 可解释性强

## 八、方法优缺点深度评析

### 8.1 优点

1. **全面优化**：同时加速视觉和语言两个阶段
2. **协同效应**：考虑耦合关系，效果更好
3. **性能保持**：联合剪枝比单独剪枝性能更优
4. **可解释性**：耦合关系清晰易懂
5. **灵活性**：可独立调整两个维度的比例

### 8.2 缺点

1. **优化复杂度**：迭代优化计算成本高
2. **超参数敏感**：需仔细设置γ和迭代次数
3. **静态剪枝**：推理时不自适应
4. **耦合估计**：梯度相关性估计可能不准确

### 8.3 潜在改进方向

1. **端到端联合优化**：替代交替优化
2. **动态耦合**：推理时动态调整
3. **多模态扩展**：扩展到更多模态
4. **任务感知**：根据任务调整耦合强度

## 十、核心公式与理论推导

### 10.1 梯度相关性

$$corr(t_i, \theta_j) = \mathbb{E}_x[\nabla_{t_i} \mathcal{L} \cdot \nabla_{\theta_j} \mathcal{L}]$$

### 10.2 耦合损失

$$\mathcal{L}_{coupling} = \sum_{i,j} corr(t_i, \theta_j) \cdot m_i^t \cdot m_j^n$$

### 10.3 交替优化收敛性

在凸条件下，交替优化可以收敛到全局最优或局部最优。
