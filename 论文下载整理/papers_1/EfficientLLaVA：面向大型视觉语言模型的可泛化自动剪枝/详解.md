# EfficientLLaVA：面向大型视觉语言模型的可泛化自动剪枝 — 详解

**EfficientLLaVA: Generalizable Automatic Pruning for Large Vision-Language Models**

---

## ★ Insight（关键洞察）

1. **跨任务泛化**：首次提出可泛化到未见任务的视觉token剪枝方法，通过多任务训练和对比学习增强预测器的泛化能力

2. **架构无关设计**：提取与模型架构无关的统计特征（激活统计、梯度统计、注意力权重），实现跨架构迁移

3. **即插即用预测器**：训练轻量级预测器可应用于各种VLM架构，无需修改原模型或重新训练

4. **输入自适应剪枝**：根据输入图像的统计特性动态调整剪枝决策，而非使用固定阈值

5. **多任务联合训练**：在多个任务上联合训练预测器，相比单任务训练具有更好的泛化性能

---

## 一、论文核心故事线（先读这里）

**一句话总结**：提出EfficientLLaVA，通过多任务训练和对比学习训练可泛化的预测器，实现跨架构跨任务的自动剪枝，在2-4倍加速下保持96%以上性能。

**核心创新点**：
- 架构无关的统计特征
- 多任务联合训练
- 对比学习增强泛化
- 即插即用预测器

**技术路径**：特征收集 → 归一化处理 → 多任务训练 → 泛化预测

## 二、研究背景与动机

### 2.1 VLM效率优化的必要性

当前VLM面临的挑战：
- **计算密集**：LLaVA-1.5-7B推理需数十GB显存
- **延迟高**：单张图像推理延迟可达数百毫秒
- **成本高**：大规模部署成本可观

### 2.2 现有方法的泛化问题

| 方法 | 训练任务 | 新任务性能 | 说明 |
|------|---------|-----------|------|
| TokenPanding | VQAv2 | 下降明显 | 任务特定 |
| 早期压缩 | 单任务 | 迁移困难 | 架构特定 |
| **EfficientLLaVA** | **多任务** | **保持** | **可泛化** |

### 2.3 核心动机

设计可泛化的剪枝方法：
- **跨任务泛化**：训练任务之外的新任务也能受益
- **跨架构泛化**：不同VLM架构使用同一预测器
- **即插即用**：无需修改原模型
- **输入自适应**：根据输入动态调整

## 三、核心问题分析

### 3.1 问题定义

给定视觉token序列 $T = [t_1, ..., t_n]$ 和预测器 $g$，学习泛化的 $g$：

$$\min_{g} \sum_{task \in T_{train}} \mathcal{L}_{task}(f(T[g(T)]))$$

约束：在 $T_{test}$ 上泛化性能下降 < 4%

### 3.2 技术挑战

**挑战1：任务特异性**
- 不同任务对视觉信息的依赖不同
- 分类任务：关注全局特征
- 检测任务：保留空间位置
- **解决方案**：多任务训练

**挑战2：架构差异**
- 不同VLM的token维度不同
- CLIP vs EVA-CLIP vs SigLIP
- **解决方案**：归一化特征

**挑战3：输入变化**
- 不同图像的内容复杂度不同
- 简单图像：更多token可剪枝
- 复杂图像：需保留更多token
- **解决方案**：输入自适应

## 四、方法详解（含公式）

### 4.1 统计特征收集

#### 4.1.1 激活统计

$$s_i^{act} = [mean(t_i), std(t_i), max(t_i), min(t_i)]$$

#### 4.1.2 梯度统计

$$s_i^{grad} = [mean(\nabla t_i), std(\nabla t_i)]$$

#### 4.1.3 注意力统计

$$s_i^{att} = [attention(t_i), importance(t_i)]$$

#### 4.1.4 组合特征

$$s_i = concat(s_i^{act}, s_i^{grad}, s_i^{att})$$

### 4.2 归一化处理

#### 4.2.1 跨架构归一化

对不同架构的特征进行Z-score归一化：
$$\hat{s}_i = \frac{s_i - \mu_{arch}}{\sigma_{arch}}$$

其中 $\mu_{arch}, \sigma_{arch}$ 是该架构的均值和标准差。

#### 4.2.2 Token级别归一化

$$\tilde{s}_i = \frac{s_i - \mu_{token}}{\sigma_{token}}$$

### 4.3 预测器架构

轻量级MLP：
$$p_i = \sigma(W_3 \cdot \text{ReLU}(W_2 \cdot \text{ReLU}(W_1 \cdot \tilde{s}_i + b_1) + b_2) + b_3)$$

三层MLP，约300K参数。

### 4.4 多任务训练

#### 4.4.1 任务损失

$$\mathcal{L}_{multi} = \sum_{k=1}^{K} w_k \cdot \mathcal{L}_{task_k}$$

其中 $w_k$ 是任务权重。

#### 4.4.2 稀疏度正则

$$\mathcal{L}_{reg} = \lambda \cdot \frac{1}{n} \sum_i p_i$$

### 4.5 对比学习增强

#### 4.5.1 正负样本构建

**正样本**：同图像的重要token对
**负样本**：不同图像的token对

#### 4.5.2 对比损失

$$\mathcal{L}_{对比} = -\log \frac{exp(sim(p_i, p_j)/\tau)}{\sum_k exp(sim(p_i, p_k)/\tau)}$$

### 4.6 算法流程

```
Algorithm: EfficientLLaVA Training
Input: Multi-task dataset D, VLM f
Output: Trained predictor g

1. # 初始化
2. g = LightweightMLP()
3.
4. # 多任务训练
5. for epoch in range(N):
6.     for task_k in tasks:
7.         for x, y in D_k:
8.             # 收集特征
9.             s = collect_statistics(f, x)
10.
11.            # 归一化
12.            s_norm = normalize(s)
13.
14.            # 预测
15.            p = g(s_norm)
16.
17.            # 剪枝
18.            T_pruned = prune(T, p, target_sparsity)
19.
20.            # 计算损失
21.            L_task = task_loss(f, T_pruned, y)
22.            L_reg = λ * mean(p)
23.            L = w_k * L_task + L_reg
24.
25.            # 更新
26.            g = update(g, L)
27.
28. return g
```

## 七、与相关工作的关系

### 7.1 与SparseVLM的关系

SparseVLM是任务特定的剪枝：
- 针对训练任务优化
- 泛化能力有限
- 需针对每个任务训练

EfficientLLaVA是可泛化的剪枝：
- 多任务联合训练
- 泛化能力优秀
- 单一预测器服务多任务

### 7.2 与TokenPanding的关系

TokenPanding是可学习的合并：
- 需修改模型架构
- 需大量训练
- 仅适用于特定模型

EfficientLLaVA是即插即用的预测器：
- 无需修改模型
- 轻量级训练
- 适用于多种模型

## 八、方法优缺点深度评析

### 8.1 优点

1. **泛化能力强**：跨任务跨架构均有效
2. **即插即用**：无需修改原模型
3. **输入自适应**：根据输入动态调整
4. **多任务学习**：从多任务中学习通用特征
5. **对比学习增强**：进一步提升泛化

### 8.2 缺点

1. **训练成本**：需多任务训练数据
2. **特征收集**：需多次前向传播
3. **静态预测器**：不随推理动态调整
4. **超参数敏感**：归一化参数需仔细调整

### 8.3 潜在改进方向

1. **持续学习**：增量添加新任务
2. **元学习**：学习如何学习剪枝
3. **动态架构**：根据输入调整网络宽度
4. **联合优化**：与模型权重联合优化

## 十、核心公式与理论推导

### 10.1 Z-score归一化

$$\hat{x} = \frac{x - \mu}{\sigma}$$

其中 $\mu = \mathbb{E}[x]$, $\sigma = \sqrt{\text{Var}[x]}$。

### 10.2 对比学习InfoNCE损失

$$\mathcal{L}_{NCE} = -\log \frac{exp(sim(q, k^+)/\tau)}{\sum_{k} exp(sim(q, k)/\tau)}$$

### 10.3 多任务加权损失

$$\mathcal{L}_{total} = \sum_{k=1}^{K} \alpha_k \mathcal{L}_k$$

权重可使用不确定性加权或逆频率加权。
