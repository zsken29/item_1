# SparseVLM：面向高效视觉语言模型推理的视觉Token稀疏化 — 详解

**SparseVLM: Visual Token Sparsification for Efficient Vision-Language Model Inference**

---

## ★ Insight（关键洞察）

1. **预测器引导的剪枝范式**：首次提出训练轻量级预测器来学习识别冗余token，将剪枝决策从静态规则转变为数据驱动，显著提升剪枝精度

2. **无需训练数据约束**：通过蒸馏式的自监督训练，预测器可以从少量样本中学习，不依赖大规模标注数据

3. **即插即用设计**：预测器作为独立模块，可以应用于各种VLM架构，无需修改原模型结构

4. **动态稀疏化**：推理时根据输入图像动态决定保留哪些token，平衡效率与精度

5. **精度损失可控**：50%稀疏度下精度损失<1%，在高效压缩与性能保持间取得良好平衡

## 二、研究背景与动机

### 2.1 VLM的效率瓶颈

视觉语言模型的推理延迟分布：
- **视觉编码器**：30-40%
- **视觉token处理**：40-50%
- **语言生成**：10-20%

视觉token处理成为最大瓶颈。

### 2.2 现有方法的局限

| 方法 | 实现难度 | 精度保持 | 适用性 |
|------|---------|---------|-------|
| 固定规则剪枝 | 低 | 差 | 通用 |
| 可学习剪枝 | 高 | 好 | 特定模型 |
| 知识蒸馏 | 高 | 好 | 需教师模型 |
| **SparseVLM** | **中** | **好** | **通用** |

### 2.3 核心动机

设计一种简单有效的VLM加速方法：
- **无需修改原模型**：即插即用
- **推理时加速**：不牺牲训练效率
- **精度可控**：可接受范围内的损失
- **通用架构**：适用于各种VLM

## 三、核心问题分析

### 3.1 问题定义

给定视觉token序列 $T = [t_1, ..., t_n]$ 和目标稀疏度 $k$，学习预测器 $g$：

$$\min_{g} \mathcal{L}_{task}(f(T[g(T)])) + \lambda (n - |T[g(T)]|)$$

约束：$|T[g(T)]| \leq n(1-k)$

### 3.2 技术挑战

**挑战1：冗余度评估**
- 如何判断哪些token可剪枝？
- 静态规则可能不准确
- **解决方案**：学习预测器

**挑战2：信息保留**
- 剪枝可能丢失关键信息
- 不同任务对视觉依赖不同
- **解决方案**：端到端优化

**挑战3：推理开销**
- 预测器本身也有计算成本
- 需小于节省的计算量
- **解决方案**：轻量级设计

## 四、方法详解（含公式）

### 4.1 预测器架构

#### 4.1.1 轻量级MLP

$$p_i = \sigma(W_2 \cdot \text{ReLU}(W_1 \cdot t_i + b_1) + b_2)$$

其中：
- $W_1 \in \mathbb{R}^{256 \times d}$：第一层权重
- $W_2 \in \mathbb{R}^{1 \times 256}$：第二层权重
- $p_i \in [0, 1]$：重要性分数
- $d$：token维度（通常为1024或更高）

#### 4.1.2 参数量分析

总参数量：$256 \times d + 256$，约0.25M参数，远小于VLM本身。

### 4.2 训练目标

#### 4.2.1 主要损失

$$\mathcal{L}_{task} = \mathbb{E}_{x,y}[-\log p(y|x, T_{pruned})]$$

使用剪枝后的token进行任务预测，计算交叉熵损失。

#### 4.2.2 正则化项

$$\mathcal{L}_{reg} = \lambda \cdot \frac{|T_{pruned}|}{|T|}$$

鼓励更激进的剪枝。

#### 4.2.3 总损失

$$\mathcal{L}_{total} = \mathcal{L}_{task} + \beta \mathcal{L}_{reg}$$

### 4.3 剪枝策略

#### 4.3.1 软剪枝（训练时）

$$T'_{soft} = T \odot \sigma(p)$$

使用sigmoid函数软化剪枝决策，便于梯度反传。

#### 4.3.2 硬剪枝（推理时）

$$T'_{hard} = T \odot \mathbb{1}[p > \tau]$$

使用阈值或Top-K进行硬剪枝。

### 4.4 训练流程

```
Algorithm: SparseVLM Training
Input: VLM f, training data D
Output: Trained predictor g

1. # 初始化
2. g = LightweightMLP()
3.
4. # 训练循环
5. for epoch in range(N_epochs):
6.     for x, y in D:
7.         # 获取视觉token
8.         T = visual_encoder(x)
9.
10.         # 预测重要性
11.         p = g(T)  # shape: [n, 1]
12.
13.         # 软剪枝
14.         T_pruned = T * sigmoid(p)
15.
16.         # 前向传播
17.         logits = f(T_pruned)
18.
19.         # 计算损失
20.         L_task = cross_entropy(logits, y)
21.         L_reg = λ * mean(p)
22.         L = L_task + L_reg
23.
24.         # 更新预测器
25.         g = g - lr * grad(L, g)
26.
27. return g
```

### 4.5 推理流程

```
Algorithm: SparseVLM Inference
Input: VLM f, predictor g, image x, target ratio r
Output: Prediction y

1. # 获取视觉token
2. T = visual_encoder(x)
3.
4. # 预测重要性
5. p = g(T)
6.
7. # 选择Top-K token
8. K = int(len(T) * r)
9. indices = top_k(p, K)
10. T_sparse = T[indices]
11.
12. # 生成回答
13. y = f(T_sparse)
14.
15. return y
```

|------|---------|---------|------|---------|
| TokenPanding | token | 需要 | 好 | 中等 |
| LLM-Pruner | 权重 | 需要 | 好 | 高 |

## 七、与相关工作的关系

### 7.1 与TokenPanding的关系

TokenPanding是可学习的token合并方法：
- 需额外训练合并操作
- 改变token维度
- 可能破坏语义

SparseVLM是选择性的token剪枝：
- 仅移除低重要性token
- 保持token维度
- 可即插即用

### 7.2 与LLaVA的关系

LLaVA是开创性的VLM架构：
- 固定token数设计
- 无压缩机制
- 高计算成本

SparseVLM在LLaVA基础上：
- 保持架构不变
- 添加预测器模块
- 实现推理加速

### 7.3 与剪枝方法的关系

传统剪枝针对权重：
- 需大量微调
- 改变模型结构
- 通用性差

SparseVLM针对token：
- 轻量级训练
- 保持模型结构
- 即插即用

## 八、方法优缺点深度评析

### 8.1 优点

1. **即插即用**：无需修改原VLM架构
2. **精度保持**：50%稀疏度下损失<1%
3. **计算高效**：预测器仅有256K参数
4. **通用适用**：可应用于各种VLM
5. **可解释**：预测分数可解释token重要性

### 8.2 缺点

1. **额外模块**：需训练和部署预测器
2. **训练成本**：仍需少量训练数据
3. **静态预测器**：不随输入动态调整架构
4. **稀疏度预设**：需预先设定目标稀疏度

### 8.3 潜在改进方向

1. **动态预测器**：根据输入复杂度自适应
2. **层级稀疏化**：不同层使用不同稀疏度
3. **多任务预测器**：针对不同任务训练专用预测器
4. **与权重剪枝结合**：联合优化token和权重

## 十、核心公式与理论推导

### 10.1 预测器输出

$$p_i = \sigma(W_2 \cdot \text{ReLU}(W_1 \cdot t_i + b_1) + b_2)$$

其中sigmoid函数将输出映射到[0,1]。

### 10.2 软剪枝梯度

$$\frac{\partial L}{\partial t_i} = \frac{\partial L}{\partial t'_i} \cdot \sigma'(p_i) \cdot \frac{\partial p_i}{\partial t_i}$$

允许梯度通过剪枝决策传递。

### 10.3 稀疏度正则

$$\mathcal{L}_{reg} = \lambda \cdot \frac{1}{n} \sum_i p_i$$

鼓励保留更少的token（更高的稀疏度）。
