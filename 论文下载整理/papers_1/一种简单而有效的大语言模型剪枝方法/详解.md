# 一种简单而有效的大语言模型剪枝方法 — 详解

**A Simple and Effective LLM Pruning Method (Wanda)**

---

## ★ Insight（关键洞察）

1. **激活感知重要性**：首次提出"重要性 = 权重 × 激活"的简洁公式，无需二阶导数即可捕捉权重在激活空间的影响

2. **免微调剪枝**：无需任何微调步骤，仅需几分钟校准数据前向传播即可完成剪枝，大大降低了计算成本

3. **实现简洁性**：相比SparseGPT等复杂方法，Wanda的实现复杂度低得多，便于实际部署

4. **任务感知能力**：通过激活信息隐式捕捉任务相关性，无需显式的任务标签

5. **线性时间复杂度**：无需迭代优化，计算时间与模型规模呈线性关系

## 三、核心问题分析

### 3.1 问题定义

给定模型 $\theta$ 和目标稀疏度 $k$，生成剪枝掩码 $m$：

$$\min_{m} \mathcal{L}_{task}(f(x; \theta \odot m))$$

约束：零值元素比例为 $k$

### 3.2 显著性评估

**核心问题**：如何衡量每个权重的重要性？

传统方法：
- 幅度：$s_i = |w_i|$ — 忽视激活影响
- Fisher：$s_i = F_i$ — 计算复杂

Wanda方法：
$$s_i = |w_i| \cdot \|激活_i\|$$

### 3.3 技术选择

**为什么选择激活作为因子？**

直觉：权重对输出的影响 = 权重本身的大小 × 该权重参与计算的频率

数学表达：
$$y_i = \sum_j w_{ij} x_j$$

对 $y_i$ 的影响与 $|w_{ij}|$ 和 $x_j$ 都成正比。

## 四、方法详解（含公式）

### 4.1 线性层显著性

对于单层感知机 $y = Wx$：

$$s_{ij} = |W_{ij}| \cdot \|x_j\|_2$$

其中 $\|x_j\|_2$ 是输入激活的$L_2$范数，衡量该维度的重要性。

### 4.2 多头注意力显著性

**Query/Key/Value投影**：
$$s_q = |W_Q| \cdot \|x\|, \quad s_k = |W_K| \cdot \|x\|, \quad s_v = |W_V| \cdot \|x\|$$

**注意力头组重要性**：
$$s_{head_h} = s_q^{h} + s_k^{h} + s_v^{h}$$

### 4.3 FFN层显著性

$$y = W_2 \cdot \sigma(W_1 x)$$

**第一层**（门控激活）：
$$s_{W_1} = |W_1| \cdot \|x\|$$

**第二层**（输出投影）：
$$s_{W_2} = |W_2| \cdot \|\sigma(W_1 x)\|$$

### 4.4 整体剪枝流程

```
Algorithm: Wanda
Input: Model M, sparsity k, calibration data D
Output: Pruned model M'

1. # 前向传播，记录激活
2. activations = {}
3. for x in D:
4.     _, acts = forward(M, x)
5.     activations.append(acts)

6. # 计算显著性分数
7. scores = {}
8. for layer in M.layers:
9.     W = layer.weight
10.    act = layer.input_activation
11.    scores[layer] = |W| * ||act||

12. # 生成剪枝掩码
13. thresholds = find_kth_percentile(scores, k)
14. mask = scores < thresholds
15. M_pruned = apply_mask(M, mask)

16. return M_pruned
```

### 4.5 与其他方法的数学对比

**Magnitude剪枝**：
$$s_i = |w_i|$$

**Fisher剪枝**：
$$s_i = \mathbb{E}[(\partial \log p / \partial w_i)^2]$$

**Wanda**：
$$s_i = |w_i| \cdot \|\mathbb{E}[x]\|$$

## 七、与相关工作的关系

### 7.1 与SparseGPT的关系

SparseGPT是当前最好的LLM剪枝方法之一，采用OBS框架。但：
- 计算复杂度 O(n²d²)
- 需迭代优化
- 实现困难

Wanda继承其无需微调的特点，但大大简化了计算。

### 7.2 与Magnitude的关系

Magnitude剪枝是最简单的剪枝方法：
- 仅考虑权重大小
- 忽视激活信息
- 对LLM效果不佳

Wanda保留了其简洁性，增加了激活感知。

### 7.3 与Wanda的联系

本文方法以Wanda命名，源于"**W**eighted **A**daptive significance for LLM Compression"。

## 八、方法优缺点深度评析

### 8.1 优点

1. **简单高效**：公式简洁，易于实现
2. **无需微调**：降低计算成本
3. **快速执行**：分钟级完成
4. **任务感知**：通过激活隐式捕获
5. **可扩展**：适用于各种LLM架构

### 8.2 缺点

1. **静态评估**：基于单次前向传播的激活统计
2. **忽视二阶效应**：未考虑权重间交互
3. **稀疏度受限**：高稀疏度时性能下降
4. **激活估计偏差**：小批量可能导致估计不准确

### 8.3 潜在改进方向

1. **动态激活估计**：多次前向传播取平均
2. **层自适应稀疏度**：不同层使用不同稀疏度
3. **结合其他方法**：与量化/蒸馏联合优化

## 十、核心公式与理论推导

### 10.1 显著性分数推导

对于线性变换 $y = Wx$：

$$y_i = \sum_j W_{ij} x_j$$

$y_i$ 对 $W_{ij}$ 的敏感度（泰勒展开）：

$$\frac{\partial y_i}{\partial W_{ij}} = x_j$$

因此：
$$|y_i \text{的变化}| \propto |W_{ij}| \cdot |x_j|$$

对 $y$ 的总影响：

$$\sum_i |\Delta y_i| \propto |W_{ij}| \cdot \sum_i |x_j| \approx |W_{ij}| \cdot \|x\|_2$$

### 10.2 批激活估计

使用小批量数据 $X = \{x_1, ..., x_B\}$ 估计：

$$\|\text{激活}_j\| \approx \sqrt{\frac{1}{B} \sum_{b=1}^{B} x_{b,j}^2} = \text{RMS}(x_j)$$
