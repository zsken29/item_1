# EvoPrune：面向高效多模态大语言模型的早期视觉Token剪枝 — 详解

**EvoPrune: Early Visual Token Pruning for Efficient Multimodal LLMs**

---

## ★ Insight（关键洞察）

1. **计算图优化思维**：首次从"计算图"角度重新思考Token剪枝时机——早期剪枝可以完全避免后续层的计算，而非仅在最后阶段减少输入

2. **50%计算减少的理论基础**：如果Token在第3层被剪枝，则第4-L层的计算完全不需要该Token参与，相比晚期剪枝可节省50%+的计算量

3. **早期Token的高信息含量**：早期视觉层的Token保留了更多的原始视觉信息，虽然看似冗余度低，但实际上更"纯净"，剪枝判断更准确

4. **79.2%精度下50%计算减少**：在显著计算节省的同时保持良好性能，证明了早期剪枝策略的有效性

5. **与模型无关**：该方法利用视觉编码器的结构特性，可应用于CLIP、EVA-CLIP等各种视觉编码器

---

## 一、论文核心故事线

**一句话总结**：提出EvoPrune，在视觉编码的早期阶段进行Token剪枝，完全避免后续层的计算，在50%计算减少下保持79.2%精度。

**核心创新点**：
- 早期剪枝时机
- 计算图优化
- 高信息含量利用
- 跨架构通用

**技术路径**：计算分析 → 早期决策 → 信息评估 → 性能保持

## 二、研究背景与动机

### 2.1 传统剪枝时机的局限

| 剪枝时机 | 问题 |
|---------|------|
| 晚期剪枝 | 大部分计算已完成 |
| 中期剪枝 | 部分计算已浪费 |
| **早期剪枝** | **无计算浪费** |

### 2.2 早期剪枝的优势

如果在层$L_p$进行剪枝：
- 层$L_p+1$到$L$的计算完全避免
- 节省$(L - L_p) / L$的计算量
